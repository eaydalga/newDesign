MySQL Geliştirilmiş Uzantı

Giriş

mysqli uzantısı MySQL 4.1 ve sonrasında belirtilen işevselliğe erişmenizi sağlar. MySQL veritabanına ilişkin daha çok bilgi için http://www.mysql.com/ adresine gidin. MySQL için belgeler http://dev.mysql.com/doc/ adresinde bulunur. Buradaki belgelerin bir bölümü MySQL içinde de vardır.

Gereksinim

Bu işlevlerin var olabilmesi için PHP mysqli uzantısıyla derlenmiş olmalıdır. Not: mysqli MySQL 4.1.3 ve üzeriyle çalışmak üzere tasarlanmıştır. Daha önceki sürümler için MySQL uzantısına bakın.

Kurulum

PHP için mysqli uzantısını kurmak için, "--with-mysqli=mysql_config_path/mysql_config" ayarlama seçeneği kullanılarak derleme yapılmalıdır. Eğer mysql uzantısını mysqli uzantısıyla birlikte kurarsanız aynı istemci kitaplığını kullanmanız çelişkileri ortadan kaldırmak açısından önemlidir.

Windows Sistemlerinde Kurulum

MySQLi varsayılan olarak açık değildir. Bu nedenle php_mysqli.dll DLL php.ini içinden açılmalıdır. PHP MySQL istemci kitaplığına da erişebilmelidir. libmysql.dll dosyası windows PHP dağıtımında paket içinde vardır. Windows PATH çevresel değişkeni ile bu dosyaya erişmek için PATH içinde gereken ayarlar yapılmalıdır. Not: MySQL 5 bağlantısı için ikililerin http://dev.mysql.com/downloads/connector/php/ adresinden indirilmesi önerilir. PHP uzantılarını açarken olduğu gibi php_mysqli.dll PHP yönelticisi olan extansion_dir ile tanımlanır. PHP 5 için bu yol c:\php\ext olarak varsayılır. Not: Eğer web sunucu başlatırken şöyle bir hata olursa: "Unable to load dynamic library './php_mysqli.dll'", Bunun nedeni php_mysqli.dll ve/ya da libmysql.dll sistem tarafından bulunamadı demektir.

Eğer hiç port tanımı yoksa MYSQL_TCP_PORT çevre değişkeninin değeri kullanılır. /etc/services içindeki mysql-tcp entry değeri, derleme zamanındaki MYSQL_PORT değişmezi de kullanılabilir. mysqli.default_socket yazı dizisi

mysqli.default_host yazı dizisi
Veritabanı sunucusuna bağlanırken, bilgisayar belirtilmemişse varsayılan bilgisayar adı kullanılır. Güvenli konuma uygulanamaz.
mysqli.default_user yazı dizisi
Veritabanına bağlanırken kullanıcı adı belirtilmemişse varsayılan kullanıcı adı kullanılır. Güvenli konuma uygulanamaz.
mysqli.default_pw yazı dizisi
Veritabanına bağlanırken şifre tanımlanmamışsa varsayılan şifre kullanılır. Bu güvenli konuma uygulanmaz.

Önceden tanımlı Sınıflar

mysqli

PHP ve MySQL arasındaki bağlantıyı belirtir.

Yapıcı (Constructor)

  • mysqli - yeni mysqli nesnesi yapılandırır

Yöntemler (Methods)

  • autocommit - veritabanı düzeltmesinde otomatik onay açık mı kapalı mı olacak onu belirtir.
  • change_user - belirtilen veritabanı bağlantısının kullanıcısını değiştirir.
  • character_set_name - Veritabanı bağlantısı için varsayılan karakter takımı adını döndürürü
  • close - daha önce açılan bir bağlantıyı kapatır.
  • commit - o anki işlemi onaylar
  • connect - MySQL veritabanı sunucusuna yeni baplantı açar.
  • debug - hata yazdırma işlemini yapar
  • dump_debug_info - hata yazma bilgisini döker
  • get_client_info - istemcinin sürümünü döndürür
  • get_host_info - kullanılan bağlantı türünü döndürür
  • get_server_info - MySQL sunucusunun sürümünü döndürür
  • get_server_version - MySQL sunucusunun sürümünü döndürür
  • init - mysqli nesnesini ilkler
  • info - en son çalıştırılan sorgu hakkındaki bilgiye erişir
  • kill - Sunucuya mysql parçasını öldürmesini bildirir
  • multi_query - çoklu sorgulama yap
  • more_results - çoklu sorgulamadan kalan daha başka sonuç var mı bak
  • next_result - o an çalıştırılmış çoklu sorgulamadan sonraki sonucu okur
  • options - seçenekleri ayarlar
  • ping - sunucu bağlantısını ping'ler ya da başka bağlantı yoksa yeniden bağlantı kurar.
  • prepare - SQL sorgusu hazırlar
  • query - sorgu yapar
  • real_connect - MySQL veritabanı sunucusuna bağlantı kurmaya kalkar
  • escape_string - SQL deyiminde kullanmak üzere yazı dizisi içindeki özel karakterleri atlatır (o anki bağlantının karakter takımını dikkate alarak atlatmayı yapar)
  • rollback - o anki işlemi geri alır
  • select_db - varsayılan veritabanını seçer
  • set_charset - varsayılan istemci karakter takımını ayarlar
  • ssl_set - ssl edğişgeçlerini ayarlar
  • stat - o anki sistem durumunu alır
  • stmt_init- mysqli_stmt_prepare ile kullanılacak deyimi ilkler
  • store_result - son sorgudan sonuç takımını aktarır
  • thread_safe - program parçası güvenliğinin verilip verilmediğine bakar.
  • use_result - son sorgudan tamponlanmamış sonuç takımını aktarır

Özellikler (Properties)

  • affected_rows - önceki MySQL işleminden etkilenen satırların sayısını alır
  • client_info - MySQL istemci sürümünü yazı dizisi olarak alır.
  • client_version - MySQL istemci sürümünü sayı olarak döndürür
  • errno - en son yapılan işlev çağırmanın hata kodunu döndürür
  • error - en son yapılan işlev çağırmanın hata mesajını döndürür
  • field_count - En son sorgu için kaç kolon olduğunu döndürür
  • host_info - kurulan bağlantının türünü gösteren yazı dizisini döndürür
  • info - en son çalıştırılan sorguya ilişkin bilgiyi döndürür
  • insert_id - Son sorguda kullanılan otomatik üretilmiş tanıtımı (ID) döndürür
  • protocol_version - Kullanılan MySQL protokolünün sürümünü döndürür
  • server_info - Sunucu sürüm numarasını gösteren yazı dizisi döndürür
  • server_version - Sunucunun sürüm numarasını tamsayı olarak döndürür
  • sqlstate - Son oluşan hatanın SQLSTATE hata kodunu yazı dizisi olarak döndürür
  • thread_id - o anki bağlantının program parçası tanıtımını döndürür (thread ID)
  • warning_count - önceki SQL deyiminden üretilen urayıların sayısını döndürür

    mysqli_stmt

    hazırlanmış bir deyimi tanımlar.

    Yöntemler (Methods)

    • bind_param - hazırlanan deyime değişkenleri bağlar
    • bind_result - sonuç bellek alanı için hazırlana deyime değişkenleri bağlar
    • close - hazırlanan deyimi kapatır
    • data_seek - deyim sonuç kümesindeki rastgele satırı arar
    • execute - hazırlanan deyimi çalıştırır
    • fetch - hazırlanan deyimden bağlı değişkenlere sunucu alır
    • free_result - verilen deyim elliği için kullanılan sonuç belleğini boşaltır
    • prepare - SQL sorgusunu hazırlar
    • reset - hazırlanan deyimi temizler
    • result_metadata - metadata bilgisi için hazırlanan deyimden sonuç kümseine erişir
    • send_long_data - öbekler halinde veriyi gönderir
    • store_result - hazırlanan deyimden sonuç kümesinin tümünü tamponlar

    Özellikler (Properties)

    • affected_rows - son çalıştırılan deyiminden etkilenen satırları döndürür
    • errno - son deyim işlevinin hata kodunu döndürür
    • error - son deyim işlevinin hata mesajını döndürür
    • field_count - sonuç kümesindeki kolon sayısını döndürür
    • id - deyimin tanıtımını döndürür
    • insert_id - hazırlanan deyimde AUTO_INCREMENT kolon için yaratılan değeri döndürür
    • num_rows - sonuç kümesindeki satır sayısını döndürür
    • param_count - hazırlanan deyim için değişgeç sayısını döndürür
    • sqlstate - Deyim işlevi için SQLSTATE hata kodunun bulunduğu yazı dizisini döndürür

    mysqli_result

    veritabanına yapıtan sorgulamadan elde edilen sonuç kümesini belirtir.

    Yöntemler (Methods)

    • close - sonuç kümesini kapatır
    • data_seek - içsel sonuç göstergesini taşır
    • fetch_array - Bir sonuç satırını ilgili dizi, sayısal dizi ya da her ikisi olarak alır
    • fetch_assoc - Sonuç satırını ilgili dizi olarak alır
    • fetch_field - Sonuç kümesinden kolon bilgisini alır
    • fetch_fields - sonuç kümesinden tüm kolonların bilgisini alır
    • fetch_field_direct - Belirtilen kolon için kolon bilgisini alır
    • fetch_object - Sonuç satırını nesne olarak alır
    • fetch_row - sonuç satırını sıralı dizi olarak alır
    • field_seek - Belirtilen alanın uzaklığı olrak sonuç göstergesini ayarlar
    • free_result - Sonuç belleğini boşaltır

    Özellikler (Properties)

    • current_field - returns offset of current fieldpointer
    • field_count - returns number of fields in resultset
    • lengths - returns an array of columnlengths
    • num_rows - returns number of rows in resultset
    • type - returns MYSQLI_STORE_RESULT or MYSQLI_USE_RESULT
