PHP / Java Birleştirmesi

Giriş

PHP ve Java'yı birleştirmenin iki yolu vardır: Ya PHP Java Servlet ortamına birleştirilir (bu çok daha verimli bir çözümdür), ya da PHP içine Java desteği eklenir. Birincisi SAPI modülünün Servlet sunucusuyla arayüz kurulmasıyla sağlanır. İkincisi ise bir Java uzantısıdır.

Java uzantısı PHP'den basit ve verimli Java nesneleri yaratma ve çağırma yöntemidir. JVM, JNI kullanılarak yaratılır ve her şey işlem içinde çalışır.

Uyarı

Bu uzantı deneyseldir. Bu uzantının davranışı, işlevleri ve adları değişebilir.

Gereksinim

Bu uzantıyı kullanabilmek için bilgisayarınızda Java VM kurulmuş olmalı ve bu uzantıyla kullanılabilmelidir.

Kurulum

Bu PECL uzantısı PHP ile paketlenmemiştir.

PHP 4'de bu PECL uzantısının kaynağı ext/ yolunda bulunabilirsi. Bu işlevleri kullanabilmek için PHP "--with-java[=DIR]" kullanılarak derlenmelidir. Budara DIR JDK'nın ana kurulum yolunu gösterir. Diğer hazırlama uzantıları php-src/ext/java/README dosyasında anlatılır.

Windows kullanıcıları bu işlevleri kullanabilmek için php.ini içinden php_java.dll açarlar. PHP 4'de bu DLL PHP Windows ikilisiyle extensions/ yolunda bulunur. Bu uzantı için DLL http://pecl4win.php.net/ adresinden indirilebilir.

Not: Bu modülü Windows ortamında PHP <= 4.0.6 ile açabilmek için PATH çevresel değişkeninin jvm.dll dosyasına erişebilmesi sağlanmalıdır. Ama PHP > 4.0.6 için ek DLL gerekmez.

Çalışma Zamanı Ayarları

Bu işlevlerin davranışları php.ini ayarlarından etkilenir.

Örnekler

Örnek-1 Java örneği

<?php
// PHP içinden java.lang.System Java sınıfının bir
// kopyasını al
$system = new Java('java.lang.System');

// Özellik erişimini göster
echo 'Java version=' . $system->getProperty('java.version') . '
';
echo 'Java vendor=' . $system->getProperty('java.vendor') . '
';
echo 'OS=' . $system->getProperty('os.name') . ' ' .
             $system->getProperty('os.version') . ' on ' .
             $system->getProperty('os.arch') . ' 
';

// java.util.Date örneği
$formatter = new Java('java.text.SimpleDateFormat',
                      "EEEE, MMMM dd, yyyy 'at' h:mm:ss a zzzz");

echo $formatter->format(new Java('java.util.Date'));
?>

Örnek-2 AWT Örneği

<?php
// CGI içinde çalışması düşünülmüştür.

$frame  = new Java('java.awt.Frame', 'PHP');
$button = new Java('java.awt.Button', 'Hello Java World!');

$frame->add('North', $button);
$frame->validate();
$frame->pack();
$frame->visible = True;

$thread = new Java('java.lang.Thread');
$thread->sleep(10000);

$frame->dispose();
?>

Notlar:

  • Eğer uygun yapıcı varsa new Java() bir sınıfın kopyasını yaratır. Eğer değişgeçler geçirilmiyorsa ve yapıcı uygunsa ve java.lang.System gibi sınıflara erişimi sağlıyorsa birçok işlevselliğin açılımı durağan yöntemlerle sağlanır.
  • Bir kopyanın bir öğesine erişimde önce çekirdek özelliklere bakılır sonra toplumsal alanlar incelenir. Diğer bir deyimle print $date.time, önce $date.getTime() olarak aranır sonra $date.time olarak.
  • Bir nesneye, hem durağan hem de o anlık kopya erişiminde aynı yazım kullanılır. Eğer Java nesnesi java.lang.Class türündense, o zaman sınıfın durağan öğelerine de erişilebilir.
  • PHP uyarıları ve NULL sonuçlar başkalıklardan doğar. Uyarılar yöntemin başına "@" konarak önlenebilir. Son hatayı alamak ve silmek için aşağıdaki API'ler kullanılabilir:
    • java_last_exception_get()
    • java_last_exception_clear()
  • Fazla yüklü çözünürlük genelde iki dil arasındaki ayrıcalıkların türüne göre zor bir sorundur. PHP Java uzantısı, hangi yoğunluğun daha uygun olduğunun yakalanması gerektiğini bulan basit ama oldukça etkin bir yapıdır. Ek olarak yöntem adları küçük büyük harf duyarlı değildir. Yoğunluktan neyin seçilmesi gerektiğini arttırır. Bir kez yöntem belirlenince, değişgeçler veri kaybı da olabilecek biçimde gerekliyse ayarlanır (kayan noktalı sayılar mantıksala çevrilir).
  • PHP geleneklerinde, diziler ve karışık tablolar değiştirilerek kullanılabilir. PHP içindeki karışık tablolar yalnız tam sayılarla ya da yazı dizileriyle dizinlenebilir. Java'daki basit türlerin dizileri çok büyük olamaz. Unutmayın ki bu yapıcıların değerleri taranır. Bu nedenle bellek ve zaman açısından biraz pahalı bir çözüm olur.

Java Servlet SAPI

Java Servlet SAPI Java uzantılarında tanımlanan bir mekanizmayla PHP işlemcisinin servlet olarak çalışmasını sağlayacak biçimde hazırlanır. PHP bakış açsından bu yaklaşımın birincil yararı, servlet'leri destekleyen web sunucular JVM'leri taramada ve yeniden kullanmadan çok büyük özen göstermektedirler. Servler SAPI modülünün hazırlanma komutları php4/sapi/README dosyasında bulunur.
  • Bu kod herhangi bir servlet makinesiyle çalışacak diye varsayılmış olsa da, bugüne değin yalnız Apache'nin Jakarta/tomcat bölümü altında denenmiştir. Hata raporları, başarı öyküleri, yamalar ve başka makinelerde kodu çalıştırmak için gereken mutlulukla kabul edilir.
  • PHP çalışma yolunu sürekli değiştirme alışkanlığı olan bir uygulamadır. sapi/servlet sonunda onu (çalışma yolunu) geri alacaktır, ama PHP çalışırken servlet makinesi CLASSPATH yolundan gereken sınıfları yükleyemeyebilir. Ya da yönetim için kullanılan yol ve JSP derleme işlemleri için sınıflar yüklenemeyebilir.
