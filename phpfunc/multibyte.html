Çoklu bayt yazı dizisi işlevleri (Multibyte String Functions)

Giriş

Dünyadaki pekçok dil 9 bitlik bayt gösterilimiyle tüm yazı karakterlerini tanımlayabilirler. Daha birçok dil için 8 bitlik bayt karakter takımlarını göstermeye yetmez. Bu nedenle çoklu bayt kodlaması geliştirilmiş, 256 karakterden daha çok karakteri olan diller için kullanılmıştır.

Çoklu bayt kodlamasında (uçlarda boşlukları silme, ayırma, dilimleme vb) işlemler yapılırken karakter tabanına göre işlem yapılır. Böyle olmasa burada belirtilen işlevler çoklu bayt karakterleri algılayamaz ve saçma sonuçlar üretir.

mbstring provides multibyte specific string functions that help you deal with multibyte encodings in PHP. In addition to that, mbstring handles character encoding conversion between the possible encoding pairs. mbstring is designed to handle Unicode-based encodings such as UTF-8 and UCS-2 and many single-byte encodings for convenience (listed below). mbstring çok baytlı yazı dizileri işlevlerinin PHp içinde çok baytlı kodlamayı kullanmalarını sağlar. Buna ek olarak mbstring olanaklı iki kodlama arasında karakter kodlama çevirilimini de ele alır (UTF-8 ve UCS-2 arasında olduğu gibi).

PHP Karakter Kodlama Gereksinimi

PHP aşadıdaki kodlamalar güvenle kullanılabilir.
  • Tek baytlık kodlama
    • ASCII uyumludur (ISO646 uyumlu) Karakterler 00h ile 7fh arasındaki sınırda bulunur.
  • Çok baytlı kodlama
    • 00h ile 7fh arasındaki karakterler ASCII karşılıklarına çevrilir.
    • ISO2022 atlatma sıralamasını kullanmaz
    • 00h ile 7fh arasındaki değeri taşımayan birleşik baytların tek karakterle gösterilimini kullanmaz.
Bunlar PHP ile çalışmayacak kodlamalardır. 
    JIS, SJIS, ISO-2022-JP, BIG-5
Bu kodlamalardan biriyle yazılmış PHP kodu büyük bir olasılıkla çalışmayabilir. Özellikle kod içinde tanıtıcı olarak kullanılan kodlanmış yazı dizileri ya da değişmezler sorunu, mbstring saydam kodlama filitresi işlevleriyle giderilir. HTTP'den gelen sorgulamalarda bu çözüm önemlidir.

Not: içsel kodlamada SJIS, BIG5, CP936, CP949 ve GB18030 kullanılması teşvik edilmez. Ama taramayla, karakter kodlamasıyla ve aramayla ilgili bilginiz yeterliyse bunları kullanabilirsiniz.

Not: PHP ile bir veritabanına bağlanırsanız, veritabanı ve içsel kodlamada daha verimli olması açısından aynı karakter kodlamasını kullanmanızda yarar vardır.

PostgreSQL kullanıyorsanız, veritabanındaki kodlamanızla PHP kodlamanızda değişik karakter kodlaması kullanmanız sorun çıkartmaz. Çünkü önyüz ve arkayüzde kodlama çevirimi veritabanı için otomatik yapılmaktadır.

Kurulum

mbstring varsayılan bir uzantı değildir. Varsayılan olarak açık değildir. Modülü "configure" seçeneğiyle açabilirsiniz. Bu konuda kurulum açıklamalarına bakın. Aşağıdaki "configure" seçenekleri mbstring modülüyle ilgilidir.
  • --enable-mbstring: mbstring işlevlerini açar. Bu seçenek mbstring işlevleri için gereklidir.

    libmbfl mbstring için gereklidir. libmbfl mbstring ile paketlenmiştir. Eğer libmbfl sistemde kuruluysa, --with-libmbfl[=DIR] kurulu kitaplığın yerini göstermek için kullanılır.

    PHP 4.3.0 sürümünden sonra mbstring uzantısı basitleşmiş Çince, Geleneksel Çince, Korece, ve Rusça ile Japanca dillerini destekler

    PHP 4.3.3 ya da öncesinde bu özelliği açmak için LANG değikeçinde --enable-mbstring=LANG; kullanılır. Basitleştirilmiş Çince desteği için --enable-mbstring=cn, Geleneksel Çince desteği için --enable-mbstring=tw, Korece desteği için --enable-mbstring=kr, Rusça desteği için --enable-mbstring=ru, Japonca desteği için --enable-mbstring=ja bu amaçla kullanılan seçeneklerdir.

    Not: PHP 4.3.4 sürümüyle, libmbfl ile desteklenen tüm kodlamalar --enable-mbstring ile açılır.
  • --enable-mbstr-enc-trans : HTTP giriş karakteri kodlama çevirimi mbstring çevirme makinesini kullanır. Eğer bu olanak açılırsa, HTTP giriş karakteri kodlaması mbstring.internal_encoding içine otomatik çevirme yapar.

    Not: As of PHP 4.3.0, the option --enable-mbstr-enc-trans was eliminated and replaced with the runtime setting mbstring.encoding_translation. HTTP input character encoding conversion is enabled when this is set to On (the default is Off).
  • --disable-mbregex: Çoklu bayt desteği olan kurallı deyim işlevlerini kapatır.

HTML 4.01 tanımlamalarına göre, bir sayfaya girilen bilginin kodlaması sayfadakinden farklı olsa bile onu değiştirme izini vardır. tarayıcının kullandığı karakter kodlamasını anlamak için mb_http_input() işlevine bakın.

Tanınmış tarayıcılar HTML sayfasının kodlaması hakkında doğruya yakın tahminde bulunsalar bile, HTML başlığında Content-Type değişgeçine karakter kodlamasının değerini yazmak daha iyidir.

Örnek-1 php.ini örnekleri ayarlama

; Varsayılan dili ayarlama
mbstring.language        = Neutral; Set default language to Neutral(UTF-8) (default)
mbstring.language        = English; Set default language to English 
mbstring.language        = Japanese; Set default language to Japanese

;; Varsayılan içsel kodlamayı ayarlama
;; Not: PHP ile karakter kodlamasının çalıştığını doğrulayın
mbstring.internal_encoding    = UTF-8  ; İçsel kodlamayı UTF-8'e ayarlayın

;; HTTP giriş kodlaması çevirimi açıldı.
mbstring.encoding_translation = On

;; HTTP giriş karakteri kodlamasını ayarla
;; Not: Program kodlaması http_input ayarlarını değiştirmez.
mbstring.http_input           = pass    ; Çevirim yok
mbstring.http_input           = auto    ; HTTP girişini auto yap
         ; "auto" "ASCII,JIS,UTF-8,EUC-JP,SJIS" olarak genişletilir
mbstring.http_input           = SJIS    ; HTTP2 girişini SJIS olarak ayarla
mbstring.http_input           = UTF-8,SJIS,EUC-JP ; Sıralamayı belirle

;; Varsayılan HTTP çıktı karakteri kodlamasını ayarlar
mbstring.http_output          = pass    ; Çevirim yok
mbstring.http_output          = UTF-8   ; HTTP çıkış kodlamasını UTF-8'e ayarla

;; Varsayılan karakter kodlama tanım sırasını ayarla
mbstring.detect_order         = auto    ; Tanımlama sırasını auto olarak ayarla
mbstring.detect_order         = ASCII,JIS,UTF-8,SJIS,EUC-JP ; Sıralı belirle

;; Varsayılan değiştirilecek karakter sırasını belirle
mbstring.substitute_character = 12307   ; Unicode değerini belirle
mbstring.substitute_character = none    ; Karakteri yazma
mbstring.substitute_character = long    ; Uzun örnek: U+3000,JIS+7E7E

HTTP Girdi ve Çıktısı

HTTP giriş/çıkış karakter kodlaması çevirimi ikili veriyi de çevirebilir. Eğer HTTP giriş/çıkışında ikili veri kullanılmışsa çevirimin kontrolu kullanıcılar tarafından yapılmalıdır.

Not: PHP 4.3.2 ya da önceki sürümlerde, bu işlevsellikte bir sınırlama vardı. mbstring POST verilerde eğer "form" öğesi multipart/form-data olarak ayarlanmışsa karakter çevirimini yapmıyordu. Bu durumda gelen verinin çevirimini siz yapmak zorundaydınız. PHP 4.3.3'den başlayarak HTML "form" için "enctype" multipart/form-data olarak ayarlanmışsa mbstring.encoding_translation değeri php.ini içinde On olarak ayarlanır. POST edilen değişkenler ve yüklenen dosyaların adları işsel kodlamaya göre çevrilir. Ama çevirim sorgulama anahtarlarına uygulanmaz.
  • HTTP Girdisi PHP kodlamalarından HTTP giriş karakter çevirimini kontrol etmenin bir yolu yoktur. Bunun php.ini içinden yapılması gerekir.

    Örnek-4 php.ini içindeki giriş çevirimini kapatın

          ;; HTTP giriş çevirimini kapatın
      mbstring.http_input = pass
      ;; HTTP giriş çevirimini kapatın (PHP 4.3.0 ya da üstü için)
      mbstring.encoding_translation = Off
    PHP Apache modülü olarak kullanıldığında, bu ayarların üzerine yazma olanağı her sanal sunucu yönelticisinde vardır. httpd.conf ya da her dosya yolundaki .htaccess ile de ayarlar değişebilir.
  • HTTP Çıktısı Çıktı karakteri kodlaması çevirimi için birkaç yol vardır. Bunlardan biri php.ini kullanarak, diğeri on_start() geri besleme işlevi olarak mb_output_handler() ile ob_start() kullanarak.

    Not: PHP3-i18n kullanıcıları mbstring çıktısı çevirimi PHP3-i18n'den farklıdır. Karakter kodlaması çıktı tamponunu kullanarak çevrilir.

Örnek-5 php.ini ayarları örneği

;; Çıktı karakter kodlamasını tüm PHP sayfaları için yapar

;; Çıktı tamponunu aç
output_buffering    = On

;; çıktı çevirimini açmak için mb_output_handler ayarla
output_handler      = mb_output_handler

Örnek-6 Kodlama örneği

<?php

// Yalnız bu sayfa için karakter kodlaması çevirimini açma

// HTTP çıktı karakter kodlamasını SJIS yap
mb_http_output('SJIS');

// Tamponlamayı başlat ve "mb_output_handler" işlevini
// geri besleme işlevi olarak belirt
ob_start('mb_output_handler');

?>

İşlevin üzerine yükleme Özelliği

Verilen bir PHP uygulamasının multibyte ortamda çalışması oldukça zordur. Bunun nedeni bir çok PHP uygulamasının substr() gibi standart işlevlerle yazılmış olmasıdır. Bu tür işlevler multibyte kodlanmış yazı dizileri için elverişli değildir.

mbstring 'işlevin üzerine yüklenmesi' özelliğini destekler. Bu da multibyte farkındalığını açar ve uygulamada kodlama değişikliğine gitmeden standart yazı dizisi işlevlerinin yerine multibyte eşdeğerlerinin kullanımı sağlanır. Örneğin mb_substr() eğer işlevin üzerine yüklemesi açıksa, substr() yerine çağrılır. Bu özellik tek bayt ortamlar için yazılmış kodlamanın multibayt ortamında kullanılmasını sağlar.

İşlev yüklemeyi kullanmak için, php.ini içindeki mbstring.func_overload ayarı yapılır. Bu positif değer bit maskelerinin kombinasyonlarının belirlediği katagorileri gösterir. Örneğin değer 1 olursa mail() işlevi üzerine yükleme yapılabilir. 2 olursa yazı dizisi işlevlerine, 4 olursa kurallı deyimlerin işlevlerine yükleme yapılabilir. Örneğin bu değer 7 olursa mail, yazı dizisi ve kurallı deyim işlevleri üzerine yükleme yapılabilir. Yüklenecek işlevler aşağıda belirtilmiştir.

Japonca multi-bayt kodlama temelleri

Japonca karakterler ancak çoklubayt kodlamasıyla gösterilebilir ve çoklu kodlama standartları ortama ve amaca göre kullanılır. İşleri daha da karıştırmak için bu kodlama standartlarının biri diğerinden azıcık farklıdır. Japaonca ortamda kullanışlı olabilecek bir Web uygulaması için geliştirici bu karmaşıklığı dikkate almalıdır.
  • Bir karakter en çok altı bayt kadar olabilir.
  • Pekçok Japonca çoklu-bayt karakterleri tek baytlık karakterlerin iki katı kadar görünürler. Buna Japonca'da "zen-kaku" (tam boy) denir. diğer dar karakterlere "han-kaku" (yarım enli) denir. Grafik özellikleri onları görüntülemekte kullanılan yüzün türüne bağımlıdır.
  • Bazı karakter kodlamaları atlatmalı (escape) sıralama kullanır (ISO-2022 de tanımlandığı gibi). Bunların kod alanı 00h ile 7fh arasında olur.
  • ISO-2022-JP SMTP/NNTP ile kullanılmalıdır, ve başlıklarla giriş noktaları RFC gereksinimine göre yeniden kodlanmalıdır. Bunlar gerekli olmasa da kullanmak iyi bir fikirdir.
  • Mobil telefonlar servisleri (i-mode, Vodafone live! ya da EZweb gibi) için hazırlanan Web sayfaları Shift-JIS kullanır.
