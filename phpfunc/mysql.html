MySQL İşlevleri

Giriş

Bu işlevler MySQL veritabanı sunucusuna erişmenizi sağlar. MySQL hakkında bilgi edinmek isterseniz http://www.mysql.com/ adresine bakın.

MySQL belgeleri http://dev.mysql.com/doc/ adresindedir.

Gereksinim

PHP MySQL desteğiyle derlenirse bu işlevler kullanılabilir.

Kurulum

Derlemek için "--with-mysql[=DIR]" ayarlama seçeneğini kullanın. Burada DIR MySQL kurulum yoludur.

Buradaki MySQL uzantısı MySQL 4.1.0 ve üzeriyle uyumlu olsa da, bu sürümlerin sağladığı ek işlevsellikleri desteklemez. Bunlar için MySQLi uzantısını kullanmalısınız.

Eğer mysql uzantısını mysqli uzantısıyla birlikte kurmak isterseniz Aynı istemi kitaplığını her ikisinde de kullanarak doğabilecek çelişkilerden kurtulmuş olursunuz.

Linux Sistemi üzerinde kurulum

PHP 4

"--with-mysql" seçeneği varsayılan olarak açıktır. Bunu kapatmak için "--without-mysql" ayarlama seçeneği kullanılabilir. Eğer MySQL, kurulum yolu belirtilmeden açılırsa, PHP paketlenmiş MySQL kitaplıklarını kullanır.

MySQL kullanan diğer uygulamaları çalıştıran kullanıcılar (örneğin auth-mysql) paketlenmiş kitaplıkları kullanmamalıdırlar. Ama kullanılan kitaplıkların yolu açıkça belirtmelidirler. Bu PHP'ye çelişkileri önelemek için istemci kitaplıklarını kullanmayı zorlar.

PHP 5+

MySQL varsayılan olarak açık değildirve MySQL kitaplığı PHP ile paketli değildir. MySQL desteğini eklemek için "--with-mysql[=DIR] ayarlama seçeneğini kullanın. Başlıkları ve kitaplıkları MySQL'den indirebilirsiniz.

Windows Sistemi üzerinde kuruluş

PHP 4

PHP MySQL uzantısı PHP ile derlenmiştir.

PHP 5+

Artık MySQL varsayılan olarak açık değildir. php_mysql.dll php.ini içine yazılarak açılabilir. Aynı zamanda PHP MySQL istemci kitaplığına erişim için gerek duyar. libmysql.dşş adlı dosya Windows PHP dağıtımına eklenmiştir ve PHP'nin MySQL ile konuşabilmesi için bu dosyanın sistem çevre değişkeni PATH tarafından erişilir olması gerekir.

Herhangi PHP uzantısının açılmasında olduğu gibi (php_mysql.dll gibi), PHP yönelticisi extension_dir PHP uzantılarının bulunduğu yolu göstermelidir. Örnek extension_dir PHP 5 için c:\php\ext olarak gösterilebilir.

Not: Eğer Web sunucuyu başlatırken aşağıdaki gibi bir hata oluşursa: "Unable to load dynamic library './php_mysql.dll'" bunun anlamı php_mysql.dll ve libmysql.dll sistem tarafından bulunamamış demektir.

MySQL Kurulum Notları

Uyarı

Kırılmalar ve başlama sorunları bu uzantıyı yüklerken karşılaşılabilir. Daha çok bilgi için "uzantıyı yeniden kodlama" bölümüne bakın.

Not: Eğer latin (varsayılan) dışında bir karakter takımına gerek varsa, o zaman dışsal (paket içinden değil) libmysql yüklemelisiniz. Burada desteklenecek karakter takımı da derlenmelidir.

İşlem başına en çok sürekli bağlantı sayısını verir
mysql.max_links tamsayı
İşlem başına en çok MySQL bağlantısını verir (sürekliler dahil)
mysql.trace_mode mantıksal
Hata tarama konumu. mysql.trace_mode açılmasıyla, tablo/dizi taramasının, serbest olmayan sonuç takımlarının ve SQL-Hatalarının görüntülenmesi sağlanır (PHP 4.3.0 sürümüyle başladı)
mysql.default_port yazı dizisi
Veritabanına bağlanırken eğer port belirtilmemişse, varsayılan TCP port numarası kullanılır. Eğer varsayılan yoksa MYSQL_TCP_PORT çevre değişkeninden değer sağlanır. /etc/services dosyasındaki mysql-tcp giriş noktası ya da MYSQL_PORT değişmezi sırayla kullanılabilecek değerlerdir (bir yoksa sonrakine bakılır). Win32 ortamında yalnız MYSQL_PORT kullanılır.
mysql.default_socket yazı dizisi
Eğer başka "socket" adı belirtilmemişse, yerel veritabanına bağlanırken kullanılacak varsayılan "socket" adıdır.
mysql.default_host yazı dizisi
Eğer başka sunucu belirtilmemişse, veritabanına bağlanırken kullanılacak varsayılan sunucu adıdır. SQL güvenli konumu için geçerli değidir.
mysql.default_user yazı dizisi
Eğer başka ad belirtilmemişse veri tabanına bağlanırken kullanılacak kullanıcı adıdır. SQL güvenli konumda kullanılmaz.
mysql.default_password yazı dizisi
Eğer başak şifre girişi belirtilmemişse, veri tabanına bağlanırken kullanılacak olan şifredir. SQL güvenli konumda kullanılmaz.
mysql.connect_timeout tamsayı
Saniye türünden bağlantı zaman aşımı. Linuz ortamında bu sunucudan beklenen ilk yanıtın zaman aşımına eşittir.

Kaynak Türleri

MySQL modülünde iki kaynak türü vardır. İlki veritabanı bağlantısı tanımıdır. ikincisi sorgulamanın sonucunu tutan kaynaktır.

Notlar

Not: Pekçok MySQL işlevi son seçimlik değişgeç olarak "link_identifier" kabul eder. Eğer yoksa bağlantı php.ini içinde tanımlı varsayılan değişgeçi kullanır. Eğer o da başarılı olmazsa FALSE döndürür.

Örnekler

Bu basit örnek MySQL veritabanına bağlantının, sorgu çalıştırmanın, sonuç yazdırmanın nasıl olduğunu gösterir.

Örnek-1 MySQL uzantısı genel görünüm örneği 
<?php
// Bağlantı, veri tabanının seçimi
$link = mysql_connect('mysql_host', 'mysql_user', 'mysql_password')
    or die('bağlanamadı: ' . mysql_error());
echo 'başarıyla bağlandı';
mysql_select_db('my_database') or die('Veritabanını seçemedi');

// SQL sorgu hazırlama
$query = 'SELECT * FROM my_table';
$result = mysql_query($query) or die('Query failed: ' . mysql_error());

// Sonuçları HTML olarak yazma
echo "\n";
while ($line = mysql_fetch_array($result, MYSQL_ASSOC)) {
    echo "\t\n";
    foreach ($line as $col_value) {
        echo "\t\t\n";
    }
    echo "\t\n";
}
echo "
$col_value
\n";

// Sonuçları boşalt
mysql_free_result($result);

// Bağlantıyı kapa
mysql_close($link);
?>
