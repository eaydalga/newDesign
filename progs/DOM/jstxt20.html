Javascript Yazılım Kuralları (JS)

DOM CSS

Her öğenin stilleri hakkında

  • ICEbrowser ve Escape/Evo kayan (float) stilleri yazamaz ya da okuyamaz.
DHTML bölümünde anlatıldığı gibi DOM uyumlu tarayıcılar 'style' nesnesini destekler. 'style' nesnesi Microsoft tarafından yaratıldı ama sonra DOM Level 2 standartı oldu. Eski DOM uyarlamalarını destekleyen tarayıcılarda buna uygun hiç bir şey yoktur. Bu nedenle evrensel olarak tüm DOM tarayıcılarına eklentisi yapılmıştır. 'style' nesnesi aracılığıyla, DOM tarayıcılar öğenin izin verdiği tüm stilleri değiştirebilirler. Bu stil niteliğini okumak ve yazmakla aynı şeydir. Stiller o nitelikle kurulabildiyse okunabilirler de. Tüm stiller hem okunur hem de yazılır türdendir.

Kural çok basit. 'element.style.styleName' yazmak yeterli. Eğer stil adı bir ya da daha çok sözcükten oluşuyorsa aralarındaki '-' işaretini çıkarın ve onu izleyen kelimenin ilk harfini büyük harf yapın. Örneğin 'background-color' yerine 'backgroundColor' stil adı olarak kullanılır. Bir değer atanacaksa yazı dizisi biçiminde olmalıdır. Diğer bilgi türleri de yazı dizisine çevrilip saklanır. Birçok tarayıcı stilin değerine null yapınca silinmesini sağlasa da bir başkası bunu yazı dizi olarak algılayabilir ve istenmeyen sonuçlar doğurabilir. Bir stili silmek isterseniz değerini boş yazı dizisi yapmak yeterlidir.

Tek tuhaf stil 'float', birçok programlama dilinde saklı sözcüktür (Javascript de dahil). Sonuç olarak bu stil cssFloat (Internel Explorer ile styleFloat) kullanılarak kurulmalıdır. Sonuçta her ikisini de kurun. Tarayıcı birini kullanacaktır.

W3C DOM öğelerin stillerini değiştirmek için değişik bir yol sunar. Burada class niteliği aşağıdakini kullanarak kaldırılabilir: 
element.className = '';
Ve şu biçimde değiştirilebilir: 
element.className = 'newClassName';

uygulanan öğe stillerini okumak

  • getComputedStyle ve currentStyle Safari 2-, Konqueror 3.4- ya da Tkhtml Hv3 tarafından desteklenmez.
CSS stil sayfaları bir sayfaya uygulandığı zaman, öğeler bir çok stil kurallarının hedefi olabilir ve çapacılarından değişik stil kuralları alabilir. Tarayıcının iç stil sayfalarından da bazı varsayılan stilleri alabilir. DOM kuralları 'window.getComputerStyle' yöntemiyle bu tür stillerin neler olduğunu belirtir. Opera 7+, Mozilla, iCab 3+, Konqueror 3.5+, ve Safari 3+ bunu destekleyen tarayıcılardır. Internet Explorer ayrıca bir öğenin 'currentStyle' özelliğini de verir. Opera 9+ ve iCab 3 de aynı desteği vermektedir.

'getComputedStyle' iki parametre ister. İlki öğeye erişim, ikincisi ise ('before', 'after', 'first-line') gibi yalancı öğeler ve öğe yerine null gibi değerlerdir... 'currentStyle' öğenin kendi stillerine erişilmesine izin verdiğine göre burada yalnız ondan söz edilecektir. Tarayıcılardan dönen değerler hep aynıdır. Ancak bazılarında ufak tefek değişiklikler vardır. Örneğin tablonun eni CSS ile atanmış olabilir ama, içindekiler sığsın diye daha da genişletilmiş olabilir. Genişletilmiş olsa bile 'currentStyle' eski enin değerini döndürse de 'getComputedStyle' ise genişletilmiş değeri döndürür. 
var oElement = document.getElementById('mydiv'), oColor;
if( window.getComputedStyle ) {
  oColor = window.getComputedStyle(oElement,null).color;
} else if( oElement.currentStyle ) {
  oColor = oElement.currentStyle.color;
}
Kullanılabilir gerçek değerlerin dışında bazı stil değerleri 'auto', 'normal', or 'inherit' gibi varsayılan değerlerdir. Tarayıcılar değerleri varolan herhangi bir birimle döndürebilir. Örneğin, CSS uzunluğu 'em' biriminden belirtse de dönen değer 'px' biçiminde olabilir.

'currentStyle' değerinin değişik sonuçlar vermesi halinde, IE 'runtimeStyle' sunar. Bu nesne neredeyse 'currentStyle' ile aynıdır ama her zaman hesaplanmış değer döndürür. Ama bu değer her zaman kullanılabilir olmaz ve sıklıkla değişik tür sonuç verir. Tanımlanmamış değerler varsayılan yerine boş yazı dizisi olur. Bunu dikkatli kullanın ve eğer sonuç veriyorsa kullanın.

Bazı öğelerin offset, client ve scroll değerlerini bilmek de yararlı olabilir. Bunlar bir öğenin özellikleri olarak vardır ve pixel birimiyle değerlerini verir. Bunun anlamı CSS değeri 'auto' olsa bile gerçek boyutları verir olmasıdır. Satır içi öğeler için çok güvenilir değildir ve yalnız blok öğelerde ya da benzeri görüntüleme biçimlerinde kullanılmalıdır. Tarayıcılar arasından oldukça sorunsuz çalışır.
offsetWidth ve offsetHeight
Öğenin boyutları sınırları dışından hesaplanır (IE quirks/dolambaçlı konumunda doldurmanın -padding- içinden hesaplanır).
clientWidth ve clientHeight
Öğenin boyutları sınırların içinden hesaplanır ve kaydırma çubuğu çıkarılır.
scrollWidth ve scrollHeight
Öğenin boyutları kaydırma çubuğu yoksa ve boyut içinde öngörülmemişse elde edilir (içeriğin boyutları artı doldurma -padding- toplam boyutu verir)
scrollLeft ve scrollTop
Öğenin kaydırıldığı uzaklık/mesafe.

Stil sayfalarını yeniden yazma

DOM stil sayfalarının değiştirilmesine de izin verir. Bazı tarayıcılar stil sayfalarının değiştirilmesine izin verir. Bazıları yaratılmasına ve silinmesine de izin verir.

Bir stilsayfasının href bağını değiştirme

Stil sayfalarının href özelliği bazı tarayıcılarda okunabilir. Opera 7+, Mozilla, Konqueror, Safari ve Internet Explorer 5+ link öğesindeki href değerini setAttribute ile değiştirir ve Internet Explorer 4+, styleSheets[index] ile değerini değiştirir.

Stil sayfalarının yerini değiştirme

  • IE 4+ Win, IE 5+ Mac, Gecko (Mozilla/Firefox/Netscape 6+), ve Opera 9+ bunu destekler
  • KHTML/WebKit (Konqueror 3+/Safari/OmniWeb 4.5+) ve ICEbrowser doğru desteklemez.
  • Mac üzerindeki IE 4 neredeyse doğru destekler.
  • Opera 7+ ve iCab 3 değişik bir yazım kuralıyla destekler.
  • KHTML/WebKit değişik bir yazım kuralıyla destekler.
  • iCab 3 üretilen içeriğe ve HTML öğesinin rengine uygulanırken önerilen stil sayfaları kapalıysa, hata yapar ve durur.
Bazı tarayıcılarda mantıksal disabled özelliği kullanılarak stil sayfaları kapatılır/açılır. Bazı tarayıcılar 'title' niteliği tanımlanmışsa, yalnız kapatılmış özelliklerin değiştirilmesine izin verirler. Tarayıcılar arası en iyi çözüm için 'title' niteliği tanımlanmışsa, aşağıdaki kodlama stil sayfalarında kapatılma / açılma denemesini yapar.

disabled özelliğinin en iyi kullanımı, okuyucularınızın sitenin görünüşünün değişmesine izin vermektir. Bunu kendi kişisel zevklerine ve erişilebilirlik gereksinimlerine uygun yapabilmelidirler. Bu da onların değişik stil sayfaları arasından seçim yapıp yerine kullanabilme olanağı kazanmalarını sağlar. Çoğunlukla bu teknik, kullanıcı sayfadan çıkarken beğendiği stil sayfalarının çerezlerde saklayarak sağlanır (unload olayını kullanarak). Daha sonra (onload olayında) aynı değeri kullanarak sayfa görüntülenir.

HTML kurulumu

Bunu yapabilmek için belgenizi aşağıdaki gibi kurun:
İlk olarak tüm evrensel stil tanımlarıyla stil sayfanızı aşağıdaki gibi yaratın: 
<link rel="stylesheet" type="text/css" href="all.css">
Hiç 'title' niteliği olmadığından 'disabled' olmayacaktır. İstemiyorsanız bu stil sayfasını yaratmak zorunda değilsiniz. Ama isterseniz, bunun içinde kullandığınız tüm stiller daha sonra başkasıyla değiştirseniz bile bütün stil sayfalarında da kullanılacaktır. Varsayılan değiştirilebilir stil sayfası aşağıdaki gibi kurulur: 
<link rel="stylesheet" type="text/css" href="default.css" title="Default">
Bu stil sayfalrını 'disable/enable' yapamayan tarayıcılar tarafından da kullanılır ve varsayılan görüntü iyidir. Bunu kullanarak diğer seçeneklerin de kurulumunu yapabilirsiniz: 
<link rel="alternate stylesheet" type="text/css"
    href="extraPretty.css" title="Pretty">
<link rel="alternate stylesheet" type="text/css"
    href="bigFont.css" title="Big Font">
<link rel="alternate stylesheet" type="text/css"
    href="contrast.css" title="High Contrast">
Bu stil sayfaları 'rel' niteliği için 'alternate stylesheet' olarak kurulduğundan küçük ve eski tarayıcılar bunları hiç yükleyemeyeceklerdir. Ve değiştirme yeteneği olmayan tarayıcılarda bu stil sayfaları hiç olmayacaktır. Stil sayfalarını değiştirebilen tarayıcılardaysa, varsayılan olarak görüntülenecektir. stil sayfalarını değiştirmekse tarayıcının görüntüleme menüsüyle olur.

Opera 7+, Mozilla, Konqueror 3+ ve iCab 3+ kullanıcının seçenekli stil sayfalarını görüntüleme menüsünden değiştirmeye olanak verirler. Ama bu kurulum tarayıcılar tarafından hatırlanmadığı için (Konqueror hariç) sitedeki her sayfada kendi yapmalıdır. Internet Explorer ve Safari gibi diğer 'iyi' taaryıcılar kullanıcıların stil sayfalarını değiştirmelerine izin vermezler, DOM kullanılılarak elle değiştirilmesi gerekir. DOM tekniği ya da view menüsünü kullanırken aynı başlık olması durumunda aynı stil sayfaları olması anlamına gelir ve her ikisi de değiştirilmiş olur.

Görüntüleme menüsü başlıkları aynı olan stil sayfalarını seçme olanağı verir. Eğer çeşitli kombinasyonları dikkate almak isterseniz, ya da aynı sayfayı birkaç kez dahil etmek isterseniz değişik başlıklar kullanmalısınız. Bu temel stil sayfaları değiştirme kodlamaları sorunlara neden olabilir.

Stil sayfalarının 'disabled' özelliği ve 'title' özelliği

document.stylesheets

Microsoft 'document.styleSheets birikimiyle ilk ortaya çıkandır. W3C bunun iyi bir fikir olduğunu ileri sürdü ve DOM Level 2 CSS kurallarına aldı. Tüm stil sayfaları bu birikimde bulunur ('rel' niteliği 'alternative stylesheet' olmasa bile). Bu birikimin 'disabled' özelliğini kurmak stilsayfasını kapatır ya da açar ve 'title' niteliği javascript özelliği olarak bulunur.
  • Internet Explorer 4+ Win ve 5+ Mac document.styleSheets destekler (doğru olarak).
  • Internet Explorer 4 Mac document.styleSheets destekler ama 'title' niteliğini sunmaz. Yerine style ya da link öğelerinin title niteliği kullanılmalıdır document.styleSheets[0].owningElement.title.
  • Gecko (Mozilla/Firefox/Netscape 6+) document.styleSheets destekler.
  • Opera 9+ document.styleSheets destekler.
  • Konqueror ve Safari [KHTML/WebKit] document.styleSheets doğru olmayarak belirli bir anda açık olan stil sayfalarının toplanmasını sağlar. Stil sayfaları oradan kapatılamaz ve 'title' özellikleri yoktur.
  • ICEbrowser kapatılacak/açılacak birikimi ve yalnız 'rel' niteliği 'sytlesheet' olanları alır ama 'alternate sytlesheet' olanları almaz. Bu da stil sayfaları değiştirme amacına uygun değildir ve W3C HTML kurallarıyla uyuşmaz. ICEbrowser kullanıcıları varsayılan stil sayfasına takılı kalacaktır, değiştirme girişimi sonucu değiştirmeyecektir.
  • NetFront birikimi sunar ama içi hep boştur...
  • Opera 7-8 ve iCab 3 birikimi sunmaz bile.

Biçimlere erişim

"document.styleSheets" eşdeğeri uygun LINK ve STYLE biçimlerinin erişimiyle sağlanır. 'disabled' özelliği okunur ve değiştirilebilir ve 'title' niteliği de sunulmuştur.
  • Internet Explorer 4+ Win bu tekniği destekler ama 'disabled' niteliği Javascript ile kurulmalıdır.
  • Internet Explorer 4+ Mac bu tekniği doğru olarak destekler.
  • Gecko (Netscape 6+, Mozilla vb.) bu tekniği doğru olarak destekler.
  • Opera 9 bu tekniği doğru olarak destekler.
  • Konqueror ve Safari [KHTML/WebKit] bu tekniği destekler, fakat 'disabled' özelliği Javascript tarafından kurulunca dek yanlış sonuç verir.
  • Opera 7-8 ve iCab 3+ bu tekniği destekler ama Javascript 'disabled' özelliğini kurmalıdır. Görüntüleme menüsüyle yapılan değişiklikler yansımaz.
  • ICEbrowser bu tekniği desteklemez.
Buradan şu sonuç çıkarılabilir:
  • Opera 7-8 ve iCab 3, link ve style biçimlerini kullanma tekniğinde bir stil sayfası seçilmeden özellikleri saklanmamalıdır.
  • Aşağıdakilerden biri seçilir:
    • Opera 9+ link ve style biçimini kullanma tekniği seçilir.
    • Opera 9+ document.stylesheets birikimi kullanılmalıdır.
  • KHTML/WebKit, link ve style biçim tekniklerini kullanır. Bir stil sayfası seçilmeden özellikleri saklanmamalıdır.
  • Aşağıdakilerden biri seçilir:
    • Gecko ile link ve style biçimini kullanma tekniği seçilir.
    • Gecko'da document.stylesheets birikimi seçilir.
  • Aşağıdakilerden biri seçilir:
    • document.all.tags kullanan IE'de link ve style biçimleri seçilir. IE 4'de erişmek için de kullanılabilir. Bir stil sayfası seçilmeden özellikleri saklanmamalıdır.
    • IE'de document.stylesheets birikimini kullanın bu IE 4'ü de içermeyi kolaylaştırır. sahib olan öğedeki başlığı kullanmaya özen gösterin.
  • ICEbrowser ile document.stylesheets birikimini kullanın ve "alternate stylesheets" ilgi alanınızsa önemlidir.
W3C doğru yöntemin stil sayfaları olduğunu söylemektedir, ama bu ancak üç tarayıcıda çalışabildiğinden ve diğerlerinde sorun oluşturduğundan burada link ve style biçimleri kullanılır. ICEbrowser koşulu da eklenmelidir. Belki hatayı değiştirmiş olabilirler.

Kullanıcının görüntüleme menüsünü kullanıp kullanmadığının anlaşılması hangi stil sayfalarının 'onload' ve 'onunload' olayından sonra kullanıldığının kontrol edilmesidir. Eğer fark varsa, o zaman kullanıcının görüntüleme menüsünden seçim yaptığı anlaşılır (Opear 7-8 tarayıcında bu olanaksızdır. 'disabled' özelliği görüntüleme menüsündeki değişiklikleri yansıtmaz). Ayrıca changeStyle işlevinin çalışıp değişkenin içeriğini değiştirdiği de kontrol edilmelidir.

Doğru tekniği kullanmak isteyebilirsiniz, öyle de olsa link ve style biçimleri tekniğini Opera 7-8 ve iCab tarayıcıları için kullanmalısınız (if(!document.styleSheets) vb.), ve ayrıca tarayıcıyı da seçmelisiniz. Çünkü KHTML/WebKit link ve style biçimleri tekniğini kullanır. KHTML/WebKit, iCab ve Opera 7-8 tarayıcıları için hala kullanıcı seçimini yapmadan saklama yapmamak gerektiğini bilmeniz gerekecektir. Ve IE 4 Mac için başlığın olmadığını bilerek kodlama yapmanız gerektiğini bilmelisiniz.

NOT: getElementsByTagName 'index' öğeli 'length' özellikli bir değer döndürür ve doğru bir birikim döndürmez. link ve style biçimlerinin dizilerini eklemek için array.concat yöntemi kullanılamaz, yerine birikimin içinde dolaşılmalı ve sırası geldikçe her öğeyi eklemmeliyiz. 
function getAllSheets() {
  // ICEbrowser tarayıcısının sınırlı desteğini isterseniz buradaki gibi yapın
  if( !window.ScriptEngine && navigator.__ice_version ) {
  	//IE hataları: pencere kapanırken navigator.__ice_version görürse
  	//window.ScriptEngine onu gizler
    return document.styleSheets; }
  if( document.getElementsByTagName ) {
    //DOM browsers - link ve style biçimlerini al
    var Lt = document.getElementsByTagName('link');
    var St = document.getElementsByTagName('style');
  } else if( document.styleSheets && document.all ) {
    //document.all destekleyenlerin hepsi document.all.tags desteklemez
    //ama destekleyenler stil sayfalarını değiştirmeyi de sunarlar
    //document.styleSheets (document.all.tags kontrolu hata verirse
    //IE içinde [NEDEN?!], gerçekte destekliyor olsa da)
    var Lt = document.all.tags('LINK'), St = document.all.tags('STYLE');
  } else { return []; } //daha ilkel bir tarayıcı - boş dizi döndür
  //tüm link biçimleri için ...
  for( var x = 0, os = []; Lt[x]; x++ ) {
    //rel niteliğini ara içinde 'style' var mı bak
    if( Lt[x].rel ) {
        var rel = Lt[x].rel;
    } else if( Lt[x].getAttribute ) {
        var rel = Lt[x].getAttribute('rel');
    } else {
        var rel = '';
    }
    if( typeof( rel ) == 'string' &&
        rel.toLowerCase().indexOf('style') + 1 ) {
      //işletim sistemini bağlı stil sayfalarıyla doldur
      os[os.length] = Lt[x];
    }
  }
  //tüm stil biçimlerini de ekle ve diziyi döndür
  for( var x = 0; St[x]; x++ ) { os[os.length] = St[x]; } return os;
}
function changeStyle() {
  for( var x = 0, ss = getAllSheets(); ss[x]; x++ ) {
    //tüm stil sayfaları için
    if( ss[x].title ) {
      //değiştirilebilirse stilsayfasını kapat
      ss[x].disabled = true;
    }
    for( var y = 0; y < arguments.length; y++ ) {
      //her başlığı kontrol et...
      if( ss[x].title == arguments[y] ) {
        //eğer seçilen bir başlığı varsa stil sayfasını tekrar aç
        ss[x].disabled = false;
      }
    }
  }
  if( !ss.length ) { alert( 'Tarayıcınız stil sayfalarını değiştiremez' ); }
}
