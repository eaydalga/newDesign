Ayarlama Seçenekleri

"configure" çekirdek seçeneklerinin listesi

Aşağıda tam olmayan ayarlama seçenekleri vardır. Bunlar PHP "configure" kodlamasıyla kullanılır. Unix benzeri ortamlarda PHP derlenirken belirtilir. Birçok "configure" seçeneği burada değil de kendi yerlerinde listelenir. Güncel ve tam bir liste için autoconf peşinden "./configure --help" komutunu PHP kaynak kod yoludan çalıştırın. GNU "configure" belgesini ek ayarlama seçenekleri için okumak isteyebilirsiniz. Böylece "--prefix=PREFIX" biçimindeki seçeneklerin anlamını da öğrenmiş olabilirsiniz.

Not: Bunlar yalnız derleme zamnında kullanılır. Eğer PHP'nin çalışma zamanı ayarlarını değiştirmek isterseniz, Çalışma Zamanı Ayarları adı bölüme bakın.

PHP 4 içindeki Ayarlama Seçenekleri

Not: Bu seçenekler yalnız PHP 4 içindir. Bazıları daha eski sürümlerde de vardır. Eski sürümleri derlemek isterseniz bazı seçenekler olmayabilir.

Çeşitli Seçenekler

--enable-debug
Hata inceleme simgeleriyle derleme.
--with-layout=TYPE
Kurulan dosyaların nasıl yayılacağını belirtir. Tür (TYPE) PHP ya da GNU olur.
--with-pear=DIR
DIR yoluna PEARi kur (varsayılan PREFIX/lib/php).
--without-pear
PEAR kurma.
--enable-sigchild
PHP'nin kendi SIGHLD ele alma göstergesini açma.
--disable-rpath
Ek çalışma zamanı kitaplık yollarını aramayı kapat.
--enable-libgcc
libgcc yerine kesin bağ kurması açılır.
--enable-php-streams
denemelik PHP akarlarını ekler. Kodu denemiyorsanız kullanmayın.
--with-zlib-dir[=DIR]
zlib kurulum yolunu tanımlama.
--enable-trans-sid
Saydam oturum yayılma kimliğini (id) açma. Yalnız PHP 4.1.2 ya da öncesi için geçerlidir. "trans-sid" özelliği herzaman derlenir.
--with-tsrm-pthreads
POSIX parçaçıklarını (thread) kullan (varsayılan).
--enable-shared[=PKGS]
Paylaşılan kitaplıkları hazırla [varsayılan=yes].
--enable-static[=PKGS]
Durağan kitaplıkları hazırla [varsayılan=yes].
--enable-fast-install[=PKGS]
Hızlı kurulum için biçimlendir [varsayılan=yes].
--with-gnu-ld
C derleyiciyinin GNU ld kullanıdığı düşünün [varsayılan=no].
--disable-libtool-lock
Kilitlemeyi kaldırı (Paralel hazırlanmayı kırabilir).
--with-pic
Yalnız PIC/non-PIC nesneleri kullanır [varsayılan=ikisini kullan].
--enable-memory-limit
Bellek sınırı desteğiyle derleme. (PHP 5.2.1 sürümüyle kullanılır - herzaman açıktır)
--disable-url-fopen-wrapper
"URL-aware" "fopen" tamamlayıcısı HTTP ya da FTP ile dosyalara erişimi sağlamayı kapatır. (PHP 5.2.5 sürümünden önce yok)
--enable-versioning
Yalnız gerekli simgeleri dışa taşır. Daha çok bilgi için INSTALL konusuna bakın.
--with-imsp[=DIR]
IMSp desteğini ekle (DIR IMSP'nin ekleme yolu ve libimsp.a yoludur). Yalnız PHP 3 için!
--with-mck[=DIR]
Cybercash MCK desteğini ekle. DIR cybercash mck hazırlama yoludur. Varsayılan değeri /usr/src/mck-3.2.0.3-linux olur. Yardım için extra/cyberlib bölümüne bakın. Yalnız PHP 3 için.
--with-mod-dav=DIR
DAV desteğini Apache'nin mod_dav modülüyle ekler, DIR mod_dav'ın kurulum yoludur (Yalnız Apache modül sürümü vardır) PHP 3 içindir.
--enable-debugger
Uzak hata ayıklama işlevleriyle derleme yapar. PHP 3 içindir.
--enable-versioning
Sürümlemek ve kapsamak yararlarını alır. Solaris 2.x ve Linux destekler. PHP 3 içindir.

PHP seçenekleri

--enable-maintainer-mode
Kullanışlı olmayan (bazen karışık olan) kuralları ve bağımlılıkları olağan kurucuya açar.
--with-config-file-path=PATH
php.ini dosyasına bakılacak dosya yolunu kurar. Varsayılanı PREFIX/lib yoludur.
--enable-safe-mode
Güvenli konumu varsayılan olarak açar.
--with-exec-dir[=DIR]
Güvenli konumdayken DIR içinde yalnız çalışabilenlere izin ver. Varsayılan yol /usr/local/php/bin olur.
--enable-magic-quotes
Varsayılan olarak sihirli tırnaklara izin ver.
--disable-short-tags
Varsayılan olarak kısa-yapı <? başlama biçimini kapat.

SAPI seçenekleri

PHP için var olan SAPI&s (Sunucu Uygulama Programı Arayüzü) listesidir.
--with-aolserver=DIR
Kurulmuş AOLserver için yolu belirtir.
--with-apxs[=FILE]
Paylaşımlı Apache modülü hazırlar. FILE Apache apxs aracının seçimlik dosya yoludur. Varsayılanı "apxs" olur. Sisteminizde kurulu olan Apache apxs sürümünü doğru belirtiğinizden emin olun ve Apache tar topunda olanı kullanmayın.
--with-apache[=DIR]
Durağan Apache modülü hazırlama. DIR Apache yolunun tepe noktasıdır. Varsayılan /usr/local/apche olur.
--with-mod_charset
mod_charset için transfer tablolarını açma (Rusca Apache).
--with-apxs2[=FILE]
Paylaşılan Apache 2.0 modülü hazırlama. FILE Apache apxs araçlarına seçimlik dosya yoludur. Varsayılan apxs olur.
--with-caudium=DIR
Caudium ile kullanılmak üzere Pike modülü için PHP hazırla. DIR Caudium sunucu yoludur. Varsayılan değeri /usr/local/caudium/server olur.
--disable-cli
PHP 4.3.0 ürümüyle var oldu. PHP'nin CLI sürümünü kapatır (bu --without-pear seçeneğini zorlar). Komut satırından PHP kullanımında daha çok bilgi vardır.
--enable-embed[=TYPE]
Gömülü SAPI kitaplığının hazırlanmasını açar. TYPE "shared" ya da "static" olur. Varsayılanı "shared" olur. PHP 4.3.0 sürümünde vardır.
--with-fhttpd[=DIR]
fhttpd modülünü hazırlar. DIR fhttpd kaynak kod yoludur. Varsayılanı /usr/local/src/fhttpd olur. PHP 4.3.0 sürümünden sonra kullanılmaz.
--with-isapi=DIR
Zeus ile kullanılmak üzere PHP'yi ISAPI modülü olarak hazırlar.
--with-nsapi=DIR
Kurulmuş Netscape/iPlanet/SunONE Web sunucuları için yolu belirtir..
--with-phttpd=DIR
Henüz bir bilgi yok.
--with-pi3web=DIR
Pi3Web ile kullanılmak üzere PHP modül olarak hazırlanır.
--with-roxen=DIR
Pike modülü olarak PHP hazırla. DIR ana Roxen yoludur. Normalde /usr/local/roxen/server olur.
--enable-roxen-zts
Zend Güvenli parçasıyla (thread) Roxen modülü hazırlar.
--with-servlet[=DIR]
servlet desteğini ekle. DIR JSDK için ana kurulum yoludur. Bu SAPI için java uzantısı gereklidir ve paylaşımlı dl olarak hazırlanmalıdır.
--with-thttpd=SRCDIR
PHP'yi thttpd modülü olarak hazırla.
--with-tux=MODULEDIR
PHP'yi TUX modülü olarak hazırla (yalnız Linux).
--with-webjames=SRCDIR
PHP'yi WebJames modülü olarak hazırla (Yalnız RISC OS)
--disable-cgi
PHP'nin CGI sürümünü kapatma. PHP 4.3.0 ile geçerlidir.
--enable-force-cgi-redirect
İçsel sunucu yeniden yönlendirmesinde güvenlik denetimini açar. Apache ile CGI sürümü çalışırsa bunu kullanmalısınız.
--enable-discard-path
Bu açılırsa, PHP CGI ikilisi güvenle web ağacının dışına taşınabilir ve insanlar da .htaccess güvenliğini delemez.
--with-fastcgi
PHP'yi FastCGI uygulamasını hazırlar. PHP 4.3.0 sürümünden sonra kaldırıldı. Yerine "--enable-fastcgi" kullanabilirsiniz.
--enable-fastcgi
Bu açıksa, CGI modülü FastCGI desteğiyle hazırlanır. PHP 4.3.0 sürümden beri kullanımdadır.
--disable-path-info-check
Eğer bu kapalıysa, /info.php/test?a=b çalışmaz. PHP 4.3.0 sürümünden beri vardır. Daha çok bilgi için Apache belgelerine bakın.
