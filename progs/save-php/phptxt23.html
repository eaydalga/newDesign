Mac OS X Kurulumu

Belgenin içeriği php-5.2.5 kaynak kod dağıtımındaki INSTALL dosyasından alınmıştır. Yeni sürümlerle ilgili değişiklikler zaman oldukça buraya eklenecektir ama resmi php sitesinden bu bilgiye erişmenizde her zaman yarar vardır.

Mac OS X üzerine PHP kurulumu

Bu bölüm Mac OS X üzerine PHP kurma ip uçlarını ve kurallarını anlatır. Mac OS X istemci ve sunucu için küçük farkları olan iki değişik kurulum vardır. Unutmayın ki Mac OS 9 ve öncesi için PHP desteği yoktur.

Paketlerin Kullanımı

Mac OS X için birkaç tane önceden derlenmiş ve paketlenmiş PHP sürümü vardır. Standart ayarlarla kurulum isterseniz bunlar size yardımcı olurlar. Ama başka özelliklerin eklenmesini isterseniz (güvenlik sunucusu, ya da başka veri tabanı sürücüsü gibi), O zaman kendi web sunucunuzu ve/ya da PHP yapınızı kendiniz kurmanız gerekir. Eğer kendi yazılımınızı kurup çalıştıramayacak gibiyseniz istediğiniz özelliklere uygun bir paket hazırlanmış mı onu aramanızda yarar vardır.

Mac OS üzerinde önceden derlenmiş ve paketlenmiş kurulumu kolay kaynaklar aşağıda bilgilerinize sunulmuştur:
  • Darwin: http://darwinports.opendarwin.org/
  • Entropy: http://www.entropy.ch/software/macosx/php/
  • Fink: http://fink.sourceforge.net/

Paketlenmiş PHP

Mac OS X sürüm 10.0.0 ile PHP Maclarda standrt oldu. Varsayılan web sunucuyla PHP açmak, Apache ayar dosyasında (httpd.conf) birkaç satırın başındaki açıklamayı kaldırmak kadar kolaydır. Burada CGI ve CLI varsayılan olarak kullanıma açıktır (Terminal programla doğrudan erişilebilir).

Aşağıdakiler hızlı bir biçimde yerel geliştirme ortamı kurmak için PHP açmayı anlatır. Her zaman PHP son sürüme göre güncellenmiş olmalıdır. Her yeni sürümde olduğu gibi hatalar ayıklanır, yeni özellikler pakete eklenir. PHP de bunlardan farklı değildir.

Standart kurulum tipi mod_php olarak belirlenir ve Mac OS X üzerinde paketlenmiş mod_php parçasını kullanıma açmak aşağıdaki adımları gerektirir:
  1. Apache ayar dosyasının yerini belirle ve aç. Genelde /etc/httpd/httpd.conf yolundadır.

    Bu dosyayı açmanın bir başka yolu da Terminal içinde Unix tabanlı bir yazı düzenleyicisi kullanmaktır (örneğin nano ve dosyanın sahibi root olduğundan sudo komutu kullanmak gerekir. 
    sudo nano /etc/httpd/httpd.conf
    Yazınca sizden şifre istenir (password:). Bazı nano komutları ^w (arama) ^o (sakla) ^x (çık) biçimindedir (^ Ctrl tuşuna karşılıktır).
  2. Yazı düzenleyicisiyle aşağıdakine benzeyen bir satırda, satırın açıklamasını kaldırın (# işaretini kaldır). Aşağıdaki satırlar peş peşe olmayabilirler. Dosyada yerlerini belirleyin: 
    # LoadModule php4_module libexec/httpd/libphp4.so

# AddModule mod_php4.c
    Yer/yol konusuna dikkat edin. İleride yine PHP kurmak gerekirse, yukarıdaki dosyalar değiştirilecek ya da işaretlenip açıklamaya dönüşecek.
  3. İstenen uzantının PHP olarak kullanılacağına emin olun (örneğin .php .html and .inc) Aşağıdaki satırın httpd.conf içinde bulunuyor olmasından (Mac Panther için doğrudur), bir kez PHP açılırsa, .php uzantılı dosyalar otomatik olarak PHP denetiminde geçer. 
    <IfModule mod_php4.c>
    # Eğer php açıksa, biz .php ve .phps dosyalarını kabul ederiz.
    AddType application/x-httpd-php .php
    AddType application/x-httpd-php-source .phps

    # Birçok kişi index.php dosyasının da çalışmasını istemesinden: 
    # biz de otomatik olarak index.php dosyasını açarız.
    <IfModule mod_dir.c>
        DirectoryIndex index.html index.php
    </IfModule>
</IfModule>
  4. DirectoryIndex varsayılan dizin dosyasını yüklerken doğru dizin dosyası kullanmalıdır. Bu httpd.conf dosyasında da vardır. Genelde index.php ve index.html kullanılır. Varsayılana göre index.php açıktır çünkü yukarıda belirtilen PHP denetimine de bağlıdır. Buna göre ayarlanmalıdır.
  5. php.ini yeri girilir ya da varsayılan değer kullanılır. Mac OS X için tipik yol /usr/local/php/php.ini ve phpinfo() çağrıldığında bu bilgiye ulaşılır. Eğer php.ini kullanılmamışsa, PHP tüm varsayılan değerleri kullanır.
  6. Belge kökü (DocumentRoot) yeri belirtilir. Bu tüm Web sayfaları için kök yoldur. Bu klasördekiler web sunucudan PHP dosyalarına kadar tarayıcıya çıkarılır. Tipik varsayılan yol /Library/WebServer/Documents olur. Ama httpd.conf dosyasında herhangi bir değere atanabilir. Ayrıca her kullanıcı için varsayılan yol /Users/yourusername/Sites biçimindedir.
  7. phpinfo() dosyası yaratın. phpinfo() PHP hakkında bilgi görüntüler. DocumentRoot klasöründe aşağıdaki PHP koduyla bir dosya olsun: 
    <?php phpinfo(); ?>
  8. Apache yeniden başlarken bu PHP dosyasını da yükleyin. Ya sudo apachectl ile çalıştırılır ya da OS X Sistem özelliklerinde "Personel Web Server" (kişisel web sunucu) çalıştırılarak başlatılır. Varsayılana göre yerel dosyaları tarayıcıya yüklemek için URL adresi http://localhost/info.php olmalıdır. Kullanıcı klasörlerindekileri yüklemek için http://localhost/~yourusername/info.php kullanılmalıdır.
CLI (ya da eski sürümlere göre CGI) adlandırılmış php olarak tanımlanır ve /usr/bin/php biçiminde çalıştırılır. Terminal açıp, PHP belgesinin komut satırı bölümünü okuyunca php -v kullanarak php sürümünün bulunduğunu anlayabilirsiniz. Bu bilgiye phpinfo() çağrılması da eklenir.

OS X Sunucu için derleme

Mac OS X Sunucu Kurulumu.
  1. En son Apache ve PHP sürümlerini alın
  2. tar toplarını açın ve Apache altındaki "configure" programını çalıştırın. 
./configure --exec-prefix=/usr \
--localstatedir=/var \
--mandir=/usr/share/man \
--libexecdir=/System/Library/Apache/Modules \
--iconsdir=/System/Library/Apache/Icons \
--includedir=/System/Library/Frameworks/Apache.framework/Versions/1.3/Headers \
--enable-shared=max \
--enable-module=most \
--target=apache
  • Eğer derleyicinin biraz verimlilik analizi yapmasını isterseniz, şu satırı da eklemelisiniz. 
    setenv OPTIM=-O2
  • Daha sonra, PHP 4 kaynak klasörüne geçin ve onu ayarlayın 
    ./configure --prefix=/usr \
    --sysconfdir=/etc \
    --localstatedir=/var \
    --mandir=/usr/share/man \
    --with-xml \
    --with-apache=/src/apache_1.3.12
    Eğer başka eklere gereksinim varsa (MySQL, GD vb) onları da burada eklemeyi unutmayın. Eklerken --with-apache seçeneği ve Apache yolunu girmek gerekecektir.
  • Artık make ve make install yazabilirsiniz. Apache kaynak klasörüne src/modules/php4 adıyla yeni bir yol daha ekler.
  • Şimdi Apache PHP 4 için yeniden ayarlanmalıdır 
    ./configure --exec-prefix=/usr \
--localstatedir=/var \
--mandir=/usr/share/man \
--libexecdir=/System/Library/Apache/Modules \
--iconsdir=/System/Library/Apache/Icons \
--includedir=/System/Library/Frameworks/Apache.framework/Versions/1.3/Headers \
--enable-shared=max \
--enable-module=most \
--target=apache \
--activate-module=src/modules/php4/libphp4.a
    Belki libmodphp4.a zaman aşımına uğradı diye mesaj alabilirsiniz. Eğer öyleyse, src/modules/php4 klasörüne gidin ve "ranlib libmodphp4.a" komutunu yazın. Sonra Apache köküne dönüp yukarıdaki ayarlamayı yeniden çalıştırın. Bu bağlantı tablosunu göncelleyecek ve hatayı kaldıracaktır.
  • php.ini-dist dosyasını bin klasörüne kopyalayın ve adını değiştirin. PHP 4 kaynak kod klasöründen: "cp php.ini-dist /usr/local/bin/php.ini" komutuyla taşımayı yapın. Ya da "local" klasörün yoksa yol "/usr/bin/php.ini" biçiminde olmalıdır.

    MacOS X İstemci için derleme

    Aşağıdaki komutlar MacOS X dahil, Apache Web sunucu için PHP modulü kurmanıza yarar. Bu sürüm MySQL ve PostgreSQL veri tabanılarını destekler.

    Uyarı

    Bunu yaparken dikkat edin Apache Sunucuyu bozabilirsiniz.

    Kurulum için bunu yapın:
    1. Bir termşnal açın.
    2. wget http://www.diax.ch/users/liyanage/software/macosx/libphp4.so.gz, ile indermeyi başlatın ve bitinceye dek bekleyin.
    3. gunzip libphp4.so.gz ile sıkıştırılmış paketi açın.
    4. sudo apxs -i -a -n php4 libphp4.so komutunu çalıştırın.
    5. sudo open -a TextEdit /etc/httpd/httpd.conf komutunu girin. TextEdit web sunucu ayar dosyasıyla açılır. Aşağıdaki iki satırı dosyanın sonuna ekleyin. 
      #AddType application/x-httpd-php .php
#AddType application/x-httpd-php-source .phps
      Baştaki # işaretlerini kaldırın (açıklama olmaktan kurtarmak anlamına gelir).
    6. Sonunda, "sudo apachectl graceful" yazarak web sunucuyu yeniden başlatın.
    PHP şimdi çalışıyordur. Sites klasörüne bir dosya koyarak deneme yapabilirsiniz. Dosya adı test.php olsun. Bu dosyanın içine <?php phpinfo() ?> satırı yazın.

    Şimdi 127.0.0.1/~your_username/test.php adresini tarayıcıda açın PHP modülleri hakkında durum tablosunu ekranda görürsünüz.

    PECL uzantıları için Kurulum

    PECL Kurulumu Girişi

    PEAR paketleme sistemiyle hazırlanmış PHP uzantılarına PECL denir. Burada PECL nasıl sağlanır ve kurulur diye anlatılmıştır.

    Bu anlatımda, /your/phpsrcdir/ yolunun PHP kaynak dağıtımının yolu olduğu varsayılmıştır. "extname" PECL uzantısının adıdır. Bu komutlar pear komutuyla benzerlik gösterir. PEAR belgelerindeki pear komut bilgisi pecl komutu için de geçerlidir.

    Kullanışlı olsun diye paylaşımlı uzantı kurulur ve yüklenir. Aşağıda anlatılan yöntem, size bir çok komut sunarak uzantıları nasıl yükleyeceğinizi anlatır ama otomatik olarak işlemleri yapmaz. Uzantılar daha sonra "extension" emriyle yüklenir. Bunu php.ini dosyasına ek yaparak ya da dl() işleviyle yapabilirsiniz.

    PHP modüllerini kurarken gereken araçların iyi/geçerli sürümlerinin bulunduğu kontrol edilmelidir (autoconf, automak, libtool vb).

    PECL uzantılarını indirme

    PECL uzantılarını indirmek için bir çok seçenek vardır:
    • http://pecl.php.net
      PECL web sitesi PHP geliştirme takımı tarafından sunulan değişik uzantılardan oluşur. Buradaki bilgi, değişiklikler, sürüm notları, gereksinim ve benzeri konulardır.
    • pecl indirme uzantı_adı (extname)
      PECL web sitesinde sunulan uzantılar listesi indirme ve kurulum için hazırıdır. Özel değişiklikler belirtilebilir.
    • CVS
      Bir çok PECL uzantıları CVS altında bulunabilir. Web tabanılı bir görüntü http://cvs.php.net/pecl/ URL adresinden sağlanabilir. CVS ortamından doğrudan indirmek için aşağıdaki komutlar kullanılabilir. Not: 'cvsread' kullanıcısının şifresi 'phpfi' olarak belirtilmiştir. 
      $ cvs -d:pserver:cvsread@cvs.php.net:/repository login
$ cvs -d:pserver:cvsread@cvs.php.net:/repository co pecl/extname
    • Windows indirmeleri
      Windows kullanıcıları için derlenmiş PECL ikilileri vardır. PECL modülleri birikimini indirmek, PECL görüntüsü ya da PECL4WIN uzantılarının DLL kitaplığını kullanırsınız. Windows altında PHP derlemek için ilgili bölüme bakın.

    Windows kullanıcıları için PECL

    PHP uzantıları DLL kitaplığında, kurulum PECL uzantıları DLL kitaplığını ilgili klasöre taşımak kadar basittir. Onları php.ini ile yüklerseniz kullanmaya başlarsınız. php.ini dosyasına aşağıdaki satırlar eklenir. 
       extension=php_extname.dll
    Bundan sonra web sunucuyu yeniden başlatın

    pecl komutuyla paylaşımlı PECL uzantılarını derleme

    PECL paylaşımlı PHP uzantılarının yaratılmasını kolaylaştırır. pecl komutunu kullanarak aşağıdakileri yapın: 
       $ pecl install extname
    Bu komut extname için kaynağı indirir, derler ve extname.so extension_dir klasörüne kurulur. Daha sonra php.ini ile extname.so yüklenebilir.

    Varsayılana göre, pecl komutu alfa ya da beta durumunda olan paketleri kurmaz. Eğer durağan paketler yoksa, beta paketi aşağıdaki komutla yüklenebilir: 
       $ pecl install extname-beta
    Özel sürümü de aşağıdakiyle kurabilirsiniz. 
       $ pecl install extname-0.1

    phpize ile paylaşımlı PECL uzantılarını derleme

    Bazen, pecl kurucusunu kullanmak seçenek değildir. Belki bir 'firewall' arkasında olabilirsiniz. Ya da kurmak istediğiniz uzantı PECL paketi olarak var olmayabilir. aynı CVS uzantısıyla yayımlanmamış paketler gibidir. Eğer siz böyle bir uzantı kurmak isterseniz, aşağı düzey kurma araçları, elle kurulumu sağlarlar.

    phpize komutu, PHP uzantıları için kurulum ortamı hazırlamakta kullanılır. Aşağıdaki örnekte uzantıların kaynakları "extname" adlı klasörde olur: 
    $ cd extname
$ phpize
$ ./configure
$ make
# make install
    Başarılı kurulum extname.so yaratır ve bunu PHP uzantısı bulunan klasöre koyun. php.ini ayarlamasında extension=extname.so satırını eklemelisiniz.

    Eğer phpize komutu sistemde yoksa, önceden derlenmiş paketler kullanılır (RPM gibi). PHP paketlerinin en uygunlarını geliştirme amaçlı kurmalısınız. phpize komutunu geçerli başlık dosyalarıyla kurun.

    phpize --help çalıştırıldığında bilginin diğer kullanımları da anlatılır.

    PHP içine PECL uzantılarını statik olarak derlemek

    PECL uzantılarını statik olarak kurmak istersiniz. Bunun için uzantıların kaynak kodunu php-src/ext/ klasörüne taşımalı ve PHP kurulumuna ayarlarını yeniden çalıştırmalısınız. 
    $ cd /your/phpsrcdir/ext
$ pecl download extname
$ gzip -d < extname.tgz | tar -xvf -
$ mv extname-x.x.x extname
    Bu aşağıdaki klasörde son bulur: 
       /your/phpsrcdir/ext/extname
    Buradan PHP ayarlama kodunu zorla yeniden çalıştırır ve PHP normal olarak kurulur: 
    $ cd /your/phpsrcdir
$ rm configure
$ ./buildconf --force
$ ./configure --help
$ ./configure --with-extname --enable-someotherext --with-foobar
$ make
$ make install
    Not: 'buildconf' çalıştırmak için gerekli olan kodlama autoconf 2.13 ve automake 1.4+ (daha yeni autoconf çalışabilir ama desteklenmez).

    uzantılara bağımlı olarak --enable-extname ya da --with-extname kullanımı değişir. Tipik olarak dış kitaplara gereksinimi olmayan uzantılar --enable ister. Emin olmak için buildconf komutundan sonra şunu çalıştırın: 
       $ ./configure --help | grep extname

    Çalışma zamanı Ayarları

    Ayar (configuration) dosyası

    Ayar dosyası (php.ini) PHP başlarken okunur. PHP yazılımı sunucu modül sürümünde bu ilk başlanırken okunur. Ama CGI ve CLI sürümlerinde her çağrılmada okunur.

    php.ini aşağıdaki yerlerde sırayla aranır.
    • SAPI modülü özel yerleşimi (Apache 2 içinde PHPIniDir klasörüdür. CGI ve CLI komut satırı için -c seçeneğiyle belirlenir. Aynı şey NSAPI için php_ini parametresinde, THTTPD için ortam değişkeni PHP_INI_PATH içindedir).
    • PHPRC çevresel değişken PHP 5.2.0 öncesinde aşağıdaki "registry key" işleminden sonra kullanılır
    • PHP 5.2.0 için aşağıdaki registry yerleşimi sırayla şöyle aranır:
      HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x.y.z\IniFilePath,
      HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x.y\IniFilePath ve
      HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x\IniFilePath,
      burada x, y ve z PHP sürüm numaralarını belirtir.
    • HKEY_LOCAL_MACHINE\SOFTWARE\PHP\IniFilePath (Windows Registry yerleşimi)
    • O anda çalışılan klasör (CLI hariç)
    • Web sunucu yolu (SAPI modüller için) ya da PHP klasörü (aksihalde Windows içinde)
    • Windows klasörü (C:\windows ya da C:\winnt) (Windows için), ya da --with-config-file-path derleme zamanı seçeneği
    Eğer php-SAPI.ini varsa (SAPI burada kullanılan SAPI anlamına gelir. örneğin php-cli.ini ya da php-apache.ini) php.ini yerine kullanılır. SAPI adı php_sapi_name() ile belirlenir.

    Not: Apache web sunucu başlarken yolu kök yapar ve PHP yazılımının php.ini dosyasını kök dosya sisteminde okumasını sağlar (eğer varsa).

    php.ini emirlerinin ele aldığı gibi, uzantıların kendi sayfaları içinde anlatıldığı görülür. Ekte çekirdek emirlerin listesi gerekir. Büyük bir olasalıkla PHP emirleri belgelerde vardır. Tüm emirleri öğrenmek için iyi açıklamalarla dolu php.ini dosyasına bakın.

    php.ini örneği 
    ; satırdaki tırnakla yazılmamış noktalı virgül dikkate alınmaz
[php] ; bölüm işaretleyicileri (köşeli parantez içindeki yazı) de dikkate alınmaz
; Mantıksal değerler aşağıdaki gibi olur:
;       true, on, yes
; ya da false, off, no, none
register_globals = off
track_errors = yes

; yazı dizilerini çift tırnak içinde tanımlayabilirsiniz
include_path = ".:/usr/local/lib/php"

; ters kesme diğer karakterler gibi algılanır 
include_path = ".;c:\php\lib"
    PHP 5.1.0 sürümünden beri .ini değişkenlerine .ini dosyasından erişilebilir (örnek open_basedir = ${open_basedir} ":/new/dir".

    Ayarlar nasıl değiştirilir

    Running PHP as an Apache module

    Apache modülü olarak php kullandığınızda, ayarları Apache ayarları dosyasındaki emirlerle değiştirebilirsiniz (httpd.conf ve .htaccess dosyaları) Bunu yapmak için "AllowOverride Options" ya da "AllowOverride All" emirlerini kullanırsınız.

    PHP 4 ve PHP 5 ile PHP ayarlarını değiştirebilecek Apache dosyalarında kullanılabilen pek çok Apache emirleri vardır. PHP_INI_ALL, PHP_INI_PERDIR ya da PHP_INI_SYSTEM bu yöntendirmelerin listesini verir.

    php_value name value
    değeri belirtilen emirlere atar. PHP_INI_ALL ve PHP_INI_PERDIR tipi emirlerde kullanılabilir. Daha önceki değeri silmek için değer olarak "none" kullanılır.

    Not: Mantıksal değerler için php_value kullanmayın. Yerine php_flag kullanın
    php_flag name on|off
    Mantıksal ayar emirlerini tanımlar. Yalnız PHP_INI_ALL ve PHP_INI_PERDIR tipi emirlerle kullanılır.
    php_admin_value name value
    Belirtilen emire değeri atar. Bu .htaccess dosyalarında kullanılamaz. php_admin_value biçimindeki emir .htaccess dosyasıyla ya da virtualhost (sanal sunucu) ile değiştirilemez. Daha önceki değeri silmek için "none" kullanılır.
    php_admin_flag name on|off
    Mantıksal ayarlama emiri atanır. Bunu .htaccess dosyasında kullanma olanağı vardır. php_admin_flag ile atanan her emir .htaccess ile ya da virtualhost ile değiştirilemez.
    Apache ayarları (örnek) 
    <IfModule mod_php5.c>
  php_value include_path ".:/usr/local/lib/php"
  php_admin_flag safe_mode on
</IfModule>
<IfModule mod_php4.c>
  php_value include_path ".:/usr/local/lib/php"
  php_admin_flag safe_mode on
</IfModule>
<IfModule mod_php3.c>
  php3_include_path ".:/usr/local/lib/php"
  php3_safe_mode on
</IfModule>

    Dikkat

    PHP değişmezleri PHP dışında var olmazlar. Örneğin httpd.conf içinde E_ALL ya da E_NOTICE gibi PHP değişmezlerini kullanamazsınız. Yerlerine bit maskesi değerlerini kullanmalısınız. Değişmezlerin yalnız php.ini dosyası içinde anlamı vardır.

    Windows registry ile PHP ayarlarını değiştirme

    Windows üzerinde PHP çalıştırmak isterseniz ayarlama değerleri klasör tabanında Windows registry kullanarak değiştirilebilir. Ayarların değerleri "registry key" ile saklanır. Alt anahtarllardaki "HKLM\SOFTWARE\PHP\Per Directory Values" yol adlarına karşılıktır. Örneğin c:\inetpub\wwwroot klasörünün ayarlama değerleri "HKLM\SOFTWARE\PHP\Per Directory Values\c\inetpub\wwwroot" altında saklanır. Bu klasör altındaki ayarlar (ya da alt klasördekiler) ancak klasördeki kodlama çalışınca etkin olur. Bu anahtar altındaki değerler PHP ayar emirlerini ve yazı dizilerini içermelidir. PHP değişmezleri değer içinde aranmaz. Ama PHP_INI_USER altındaki değişebilen ayar değeri atanabilir. PHP_INI_PERDIR değerleri bu yolla değiştirilemez.

    PHP için diğer arayüzler

    PHP yazılımını nasıl çalıştırırsanız çalıştırın, bazı değerleri init_set() ile çalışırken değiştirebilirsiniz. Eğer ayarlar listesinin tümüyle ilgilenirseniz ve sisteminizdeki değerlerini öğrenmek isterseniz phpinfo() işlevini çalıştırmalısınız. O anki özel ayarları okumak için ini_get() ya da get_cfg_var() işlevlerini kullanabilirsiniz.
