PHP Hyperlink Preprocessor

11 Ocak 2008

İlk Anlatım

Burada kısa bir anlatımla PHP kodlamasının özelliklerinden söz edilmiştir.

PHP kullanımıyla ilgili en önemli özellik, dinamik web sayfaları yaratılabilmektir. PHP web sayfaları HTML gibidir. Aynı biçimde aynı programlarla değiştirilebilir.

PHP dosyalarının uzantıları '.php' biçimindedir. Sunucu bu dosyaları önce PHP önişlemcisinden geçirir, PHP biçimleri açılıp anlamlı HTML biçimlerine ve yazılara dönüştürüldükten sonra tarayıcıya HTML olarak gönderilir. Bu yapıyı sunucu altında şöyle özetleyebiliriz. Diyelim HTML sunucu Apache olsun ve PHP modüllerle desteklensin. Bu işlem için bir de veri tabanına gerek vardır. O zaman MySQL modülleriyle beslenen sunucu dört dörtlük hazırlanmış olur. Diğer ortamlarda da benzeri altyapı hazırlanabilir. İleride bu konuyu işleyen web sayfalarında kurulumlar incelenebilir.

İlk PHP etkinleştirilmiş sayfanız

'merhaba.php' adlı bir dosyayı sunucunun belge kökünde (DOCUMENT_ROOT) olduğunu varsayalım. 
<html>
 <head>
  <title>PHP Test</title>
 </head>
 <body>
 <?php echo '<p>Merhaba Dünya</p>'; ?> 
 </body>
</html>
PHP önişlemciden geçirince, bu dosya aşağıdaki biçimde tarayıcıya gönderilir: 
<html>
 <head>
  <title>PHP Test</title>
 </head>
 <body>
 <p>Hello World</p>
 </body>
</html>
Burada PHP "echo" işlemiyle yazı dizisi yazdırılmıştır. Belki bu kadar temel bir işlem için PHP kullanmak gerekmeyebilir. Burada en önemli sorun eğer PHP dosyası bir önişlemciden çıktı vermeden geçerse, tarayıcıda indirme (download) yapılacakmış gibi davranır ve kullanıcı tüm PHP kodunu indirmiş olur.

İkinci örnek PHP özelliklerini veren 'phpinfo' işlevidir: 
   <?php phpinfo(); ?>

Daha kullanışlı birşey

$_SERVER çok özel PHP evrensel değişkenidir. Hemen her tür WEB sunucu bilgilerini içerir. 
<?php
echo $_SERVER['HTTP_USER_AGENT'];
?>
Bu örnekte kullanılan $_SERVER bir dizidir. Hatta bu diziyi oluşturan değişkene süper evrensel değişken de denir. Dizinin bir öğesine erişim örnekte görterildiği gibi olur. Kullanıcı dizi öğelerine (ya da hicrelerine) erişmek isterse ya öğenin adını ya da dizin numarasını kullanır.

Bir çok php komutu bir PHP biçimi içinde toplanabilir. Böylece PHP kodlaması öbekleri oluşabilir. Her öbek içinde birçok PHP işlemi yapılabilir.

Eğer biraz C dili yazım kurallarını biliyorsanız PHP kodlaması size mantıklı gelecektir. Ama konuyu hiç bilmiyorsanız PHP kitabının ilk birkaç bölümünü okumanızı öneririz. Aşağıdaki örnekte 'If' (Eğer) deyiminden sonra strpos() işlevi ile tanışırız. PHP içinde tanımlanmış işlevlerdendir. Bir diziyi bir başka dizi için arar. Burada biz 'MSIE' yazısını $_SERVER['HTTP_USER_AGENT'] içinde aramaktayız. Eğer eşiti bulunursa aranan yazıdaki yeriin başlangıçtan uzaklığı sayısal değer olarak bize döner. Eşiti bulamazsa FALSE döner. 'If' deyimi TRUE için kıvrımlı parantezler içindekileri işleme alır. Aksi halde bu kodlama çalışmaz. 'if,else' ikilisi 'srttoupper()' ya da 'strlen()' gibi işlevler de örnekte kullanılabilir. 
<?php
if (strpos($_SERVER['HTTP_USER_AGENT'], 'MSIE') !== FALSE) {
?>
<h3>strpos() false olmayan bir değer döndürmüştür</h3>
<p>Internet Explorer kullanıyorsunuz</p>
<?php
} else {
?>
<h3>strpos() false döndürmüş olmalı </h3>
<p>Internet Explorer kullanmuyorsunuz </p>
<?php
}
?>
Burada birşeyleri yazdırmak için echo kullanmak yerine PHP ortamından çıkıp doğrudan HTML olarak yazdırma yapılabildiği gösterilmiştir.

Formları kullanma

PHP için en güçlü olduğu konu olarak giriş formlarını ileri sürebiliriz. Her tür form öğesi otomatik olarak PHP kodlamasınca kullanılabilir. 
<form action="action.php" method="post">
 <p>Adınız: </p>
 <p>Yaşınız: </p>
 <p></p>
</form>
Yukarıdaki HTML kodlamasında gönderme (submit) yapıldığında sunucudaki kök klasörde bulunan 'action.php' kodlaması çalışmaya başlar ve sonucu yine tarayıcıda web sayfası olarak görünür. Bu örnek istemci-sunucu uygulamasının tipik bir örneğidir. Sunucuda çalışan 'action.php' aşağıdaki gibi olabilir: 
Merhaba <?php echo htmlspecialchars($_POST['name']); ?>.
Siz <?php echo (int)$_POST['age']; ?> yaşınızdasınız.
Bu örneğin en çarpıcı özelliği 'htmlspecialchars()' adlı PHP işlevidir. Giriş alanında HTML ve javascript gibi kodların olmamasını sağlar. "action.php" form bilgilerini $_POST dizisinden alır ama HTML form tanımında "method=GET" denmiş olsaydı bilgiler $_GET dizisinden alınacaktı. $_REQUEST dizisini de kullanma olanağı vardır. O zaman "method" hakkında bilgi sahibi olmak gerekmez.

Eski kodu yeni PHP ile kullanmak

Eskiden $_HTTP_*_VAR form değişkenleri için kullanılırken yeni PHP ortamında $_GET, $_POST $_COOKIE, $_SERVER, $_FILES, $_ENV, $_REQUEST, ve $_SESSION kullanılır oldu.

Eskiden olduğu gibi kayıtlı değişkenleri kullanmak olanağı yoktur. Örneğin eskiden "http://alan_adi/birsey.php?id=1" gibi bir bağdan sonra 'birsey.php' içinde "$id" doğrudan kullanılırdı. Şimdi yerine $_GET['id'] sunulmaktadır.
