İşlevler

Kullanıcının tanımladığı işlevler

Aşağıdaki gibi bir yazım kuralıyla işlev tanımı yapılabilir: İşlev kullanımını anlatan kodumsu: 
<?php
function foo($arg_1, $arg_2, /* ..., */ $arg_n)
{
    echo "Örnek işlev.\n";
    return $retval;
}
?>
Bir işlev içinde herhangi bir PHP kodu, bir işlev ya da nesne olabilir.

İşlev adları aynı etiket adları gibi tanımlanır. Geçerli işlev adı bir harf ya da altçizgiyle başlar, harfler, sayılar ve altçizgilerle devam eder. Kurallı deyimde yazılışı [a-zA-Z_\x7f-\xff][a-zA-Z0-9_\x7f-\xff]* ile gösterilir.

İpucu

PHP 3 içinde, işlevler kullanılmadan önce tanımlanmalıydı. PHP 4 içinde böyle bir zorunluluk yoktur. Ama eğer işlev aşağıdaki iki örnekte gösterildiği gibi koşullu tanımlanmışsa farklı olur.

Eğer işlev koşullu biçimde tanımlanırsa, işlevin tanımı kullanımından önce yapılmalıdır.

Koşullu işlevlere örnek: 
<?php

$makefoo = true;

/* foo() işlevini buradan çağıramayız
   çünkü daha tanımlanmamıştır
   ama bar() işlevi çağrılabilir */

bar();

if ($makefoo) {
  function foo()
  {
    echo "Program çalışması buraya gelinceye dek ben var olmam.\n";
  }
}

/* Şimdi güvenle foo() işlevini
   çağırabiliriz. Çünkü $makefoo
   'true' olarak hesaplandı */

if ($makefoo) foo();

function bar() 
{
  echo "Programın başlamasından hemen sonra var olurum.\n";
}

?>


İşlev içinde işlev örneği: 
<?php
function foo() 
{
  function bar() 
  {
    echo "foo() çağrılıncaya dek ben var olmam.\n";
  }
}

/* Şu anda bar() çağıramayız
   Çünkü daha var olmadı */

foo();

/* Şimdi bar() işlevini çağırabiliriz
   çünkü foo() işlevinin çalışması
   onu erişilebilir kılmıştır */

bar();

?>


Tüm sınıf ve işlevlerin PHP içinde evrensel kapsama alanı vardır. Bir işlevin içinde tanımlansalar bile dışarıdan çağrılabilirler. Tersi de olabilir.

PHP işlevlerin fazla yüklenmesini (overload) desteklemez. Daha önce tanımlanmış bir işlevi yeniden tanımlamak ya da tanımı silmek olanağı yoktur.

Not: İşlev adları büyük küçük harf duyarlıdır. Tanımlarındaki gibi işlevleri çağırmak iyi bir yöntemdir.

PHP 3 işlevlerin değişken sayıda parametreleri olmasını desteklemez. Ama varsayılan parametreler desteklenir (varsayılan parametre değerlerine bakın). PHP 4 den başlayarak her ikisi de desteklenir. değişken uzunluktaki parametre listelerine ve işlev erişimlerine func_num_args(), func_get_arg(), ve func_get_args() için bakın.

PHP içinde özyinelemeli işlevler de çağrabilirsiniz. Ama özyinelemeli işlev/yöntem çağırmak 100-200 yinelenin üstünde olmamalıdır. Çünkü tampon belleği doldurur ve kodlamanın bozulmasına neden olabilir.

Özyineleme İşlevleri 
<?php
function recursion($a)
{
    if ($a < 20) {
        echo "$a\n";
        recursion($a + 1);
    }
}
?>

İşlev parametreleri

İşlevlere bilgi parametre listeleriyle geçer. Parametre listeleri virgüllerle ayrılmış PHP deyimleridir.

PHP parametrelerin değer olarak (varsayılan), adres olarak ve varsayılan parametre olarak geçirilmesini destekler. Değişken uzunlukta parametre listeleri yalnız PHP 4 ve üzerinde desteklenmektedir. Aynı etki PHP 3 altında işlev parametrelerini bir diziye aktarıp o diziyi parametre olarak göndermekle de sağlanabilir.

İşleve dizileri aktarma örneği: 
<?php
function takes_array($input)
{
    echo "$input[0] + $input[1] = ", $input[0]+$input[1];
}
?>

Adresle parametre geçirme

Varsayılan olarak işlev parametreleri değer olarak geçer (böylece parametrenin değerini işlev içinde değiştirecek olursanız, parametre değeri işlev dışında değişmemiş olur). Eğer bir işlevin parametrelerinin değerini değitirmesini isterseniz adresleriyle geçirmeniz gerekir.

Eğer bir değişkenin bir işleve adres olarak geçmesini isterseniz önüne (&) işareti koymanız yeterlidir:

İşlev parametrelerini adres olarak geçirme örneği 
<?php
function add_some_extra(&$string)
{
    $string .= ' ve birşey daha.';
}
$str = 'Bu bir yazı dizisidir, ';
add_some_extra($str);
echo $str;    // çıktı 'Bu bir yazı dizisidir, ve birşey daha.'
?>

Varsayılan parametre değerleri

Bir işlev düz parametreler için aşağıdaki gibi C++ stili varsayılan değerler tanımlayabilir:

Bir işlevde varsayılan değerlerin kullanımı örneği: 
<?php
function makecoffee($type = "cappuccino")
{
    return "Bir kap $type yapın.\n";
}
echo makecoffee();
echo makecoffee(null);
echo makecoffee("espresso");
?>
Yukarıdaki işlevin sonucu: 
Bir yap cappuccino yap.
Bir yap  yap.
Bir yap espresso yap.
PHP aynı zamanda diziler ve özel tür olan NULL ve varsayılan değer kullanılır:

Sıra dışı tür kullanımının varsayılan parametre olması örneği: 
<?php
function makecoffee($types = array("cappuccino"), $coffeeMaker = NULL)
{
    $device = is_null($coffeeMaker) ? "hands" : $coffeeMaker;
    return "Making a cup of ".join(", ", $types)." with $device.\n";
}
echo makecoffee();
echo makecoffee(array("cappuccino", "lavazza"), "teapot");
?>
Varsayılan değer değişmez deyim olmalıdır, (örneğin) değişken, sınıf üyesi ya da işlev çağırması olmamalıdır.

Varsayılan parametreleri kullanırken, varsayılan değerler varsayılan olmayan parametrelerin sağında olmalıdır. Yoksa işler beklendiği gibi çalışmaz. Aşağıdaki kodlamaya bakabilirsiniz.

İşlevlerin varsayılan parametrelerinin doğru olmayan kullanımı: 
<?php
function makeyogurt($type = "acidophilus", $flavour)
{
    return "Making a bowl of $type $flavour.\n";
}
 
echo makeyogurt("raspberry");   // beklendiği gibi çalışmaz
?>
Bu örneğin çıktısı: 
Warning: Missing argument 2 in call to makeyogurt() in 
/usr/local/etc/httpd/htdocs/php3test/functest.html on line 41
Making a bowl of raspberry .
Şimdi bununla karşılaştırın:

İşlevlerde varsayılan parametrelerin doğru kullanım örneği: 
<?php
function makeyogurt($flavour, $type = "acidophilus")
{
    return "Making a bowl of $type $flavour.\n";
}
 
echo makeyogurt("raspberry");   // works as expected
?>
Bu örneğin çıktısı: 
Making a bowl of acidophilus raspberry.


Not: PHP 5 den sonra varsayılan parametre adres olarak geçirilir

Değişken uzunlukta parametre listesi

PHP 4 ve sonrakiler kullanıcı işlevlerinde değişken uzunluktaki parametre listelerini destekler. Bu işlem func_num_args(), func_get_arg(), ve func_get_args() işlevleriyle çok kolaydır.

Özel bir yazım kuralı gerekmez. işlvelerce parametre listeleri dıştan desteklenir ve normal olarak davranır.

Dönen Değerler

Seçimli 'return' deyimiyle değerleri döndürebilirsiniz. Herhangi bir tür (listeler ve nesneler de dahil) döndürülebilir değerdir. Bu bir işlevin işlemlerini kesip çağrıldığı satıra geri dönmesini sağlar.

return() kullanım örneği 
<?php
function square($num)
{
    return $num * $num;
}
echo square(4);   // çıktı '16' olur.
?>
Bir işlevden birden çok değer döndüremezsiniz, ama bir liste döndürerek benzer sonuç elde edebilirsiniz.

birden çok değer almak için dizi döndürme örneği 
<?php
function small_numbers()
{
    return array (0, 1, 2);
}
list ($zero, $one, $two) = small_numbers();
?>
Bir işlevden adres döndürmek için hem işlev tanımında hem de dönen değerin atamasında adres işlemi kullanmanız gerekir.

Bir işlevden adres döndürme 
<?php
function &returns_reference()
{
    return $someref;
}

$newref =& returns_reference();
?>

Değişken İşlevler

PHP değişken işlevler kavramını destekler. Bunun anlamı eğer bir değişkenin peşine parantez ikilisi eklenmişse, PHP değişken neye dönüşüyorsa o adla bir işlev arar ve onu çalıştırmaya çalışır. Başka şeylerin dışında bu işlem geri dönüşlerin, işlev tabloları ve benzeri işlemlerin uyarlanmasında kullanılır.

Değişken işlevleri echo(), print(), unset(), isset(), empty(), include(), require() gibi dil yapılarıyla çalışmaz. Değişken işlevlerle benzeri kodları kullanmak için aynı işlemleri yapan kendi işlevlerinizi yazmanız gerekir.

Değişken işlev örnekleri 
<?php
function foo() {
    echo "foo() içinde
\n";
}

function bar($arg = '')
{
    echo "bar() içinde; parametre '$arg' olur.
\n";
}

// echo için kulalnılan işlev
function echoit($string)
{
    echo $string;
}

$func = 'foo';
$func();        // Bu foo() çağırır

$func = 'bar';
$func('test');  // Bu bar() çağırır

$func = 'echoit';
$func('test');  // Bu echoit() çağırır
?>
Değişken işlev özelliğini kullanarak nesne yöntemini çağırabilirsiniz.

Değişken yöntem örneği 
<?php
class Foo
{
    function Variable()
    {
        $name = 'Bar';
        $this->$name(); // Bu Bar() yöntemini çağırır
    }
    
    function Bar()
    {
        echo "Bu Bar olur";
    }
}

$foo = new Foo();
$funcname = "Variable";
$foo->$funcname();  // Bu $foo->Variable() çağırır

?>

İçsel/hazır işlevler (Internal/built-in functions)

PHP birçok işlev ve yapıyla hazır gelir. Bazi işlevler özel PHP uzantılarını isterler. Onların uzantılarla derlenmiş olması gerekir. Yoksa hata mesajı verirler. Örneğin resim işlevlerini kullanmak (imagecreatetruecolor() gibi), için GD uzantısının PHP ile derlenmiş olması gerekir. Ya da mysql_connect() kullanmak için PHP MySQL desteğiyle derlenmelidir. Her PHP sürümüyle sunulan birçok kök işlev vardır. Ayrıca birçok uzantı zaten PHP topluluğuna varsayılan olarak eklenmiştir. Eğer phpinfo() çağrılırsa ya da get_loaded_extensions() ile PHP sisteminize nelerin ekli olduğunu görebilirsiniz. Bu konu kurulumla ilgili sayfalarda anlatılmıştır.

Bir işlevin prototipini okumak ve anlamak 'işlev tanımı nasıl okunur' başlıklı sayfalarda anlatılmıştır. Bilinmesi önemli olan konu bir işlevin döndürdüğü değer mi ya da işleve geçirilen parametre değeri mi olduğudur. Örneğin str_replace() değiştirilmiş yazı dizisini döndürür ama usort() geçirilen parametrenin üzerinde çalışır. Her işlev yazısı geçirilen parametre değerleri, işlevden dönen değerleri, başarı ve hata durumundaki değerleri belirtir.

Not: Eğer geçirilen parametreler onun bekledikleri gibi değilse (yazı dizisi beklerken bir dizi geçirmek gibi) işlevin döndürdüğü değer belirsiz (undefined) olur. Bu durumda NULL döndürmesi gerekir ama bu kavram olduğundan buna bağlı kalınmamalıdır.

function_exists(), işlev erişimi, get_extension_funcs(), ve dl() sayfalarına da bakın.
