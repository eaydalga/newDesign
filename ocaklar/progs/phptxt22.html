UNIX Kurulumu

Belgenin içeriği php-5.2.5 kaynak kod dağıtımındaki INSTALL dosyasından alınmıştır. Yeni sürümlerle ilgili değişiklikler zaman oldukça buraya eklenecektir ama resmi php sitesinden bu bilgiye erişmenizde her zaman yarar vardır.

UNIX Sistemlerine Kurulum

Kendi ortamınıza özgü konuları işleme başlamadan önce bilip kendiniz için not etmenizde yarar vardır. Bu belge kişiye özel konuları içermez.

UNIX ortamlarına PHP birkaç yolla eklenebilir. Ya otomatik ayarlama ve derlene ile ya da önceden derlenip paketlenmiş tar topları kullanılarak. Burada derlenip ayarlamayla ilgili kurulum biçimi anlatılacaktır.

PHP açık kaynak yazılım olduğu için kaynak kodların bulunduğu tar topu açıldıktan sonra ilgili klasöre geçilir (Bu örnekte php-5.2.5.tar.gz açılır ve php-5.2.5 klasörüne geçilir). Burada "./configure" kullanılarak otomatik derleme ve ayarlama adımı başlatılır. Ayarlamalarda kullanılabilecek seçenekler "./configure --help" komutuyla girilir. Bir kez ayarlama yapıldı mı artık modül ve çalışabilir programlar derlenip kuruluma hazır biçime gelir.

Apache paylaşımlı modül sürümü

Burada Apache ve PHP kaynak kodlarının indirilmiş olduğu varsayılmıştır. Kurulumda işlem adımları şöyledir (Not bu bilgi PHP kaynak kodlarındaki KURULUM dosyasından alınmıştır).
  1. gunzip apache_xxx.tar.gz
  2. tar -xvf apache_xxx.tar
  3. gunzip php-xxx.tar.gz
  4. tar -xvf php-xxx.tar
  5. cd apache_xxx
  6. ./configure --prefix=/www --enable-module=so
  7. make
  8. make install
  9. cd ../php-xxx
  10. Şimdi, PHP ayarı 'configure' ile yapılır. Burada PHP seçeneklerinin hangi eklentilerle ayarlandığı gösterilmektedir. ./configure --help ile var olan seçenekler görüntülenebilir. Burada Apache ve MySQL ile basit bir ayar yapılır. 
          ./configure --with-mysql --with-apxs=/www/bin/apxs
  11. make
  12. make install

    Eğer ileride seçenekleri değiştirmek gerekirse Apache yeniden derlenmeden değişen seçeneklerle yalnız son üç adım işleme alınır.

    Ayrıca belirtilmezse 'make install' PEAR, phpsize ve PHP CLI ve başkalarının da kurulmasını sağlar.
  13. php.ini dosyasının kurulumu: 
          cp php.ini-dist /usr/local/lib/php.ini
    PHP seçeneklerini ayarlamak için .ini dosyasını düzenleyebilirsiniz. Eğer .ini dosyanızın yerini değiştirmek isterseniz adım 10 komutunda --with-config-file-path=/bir_baska/yol kullanın.

    PHP değişikliklerini anlamak için lüften herşeyi okuyun.
  14. PHP modülünü de yükleyebilmek için "httpd.conf" dosyasını değiştirin. LoadModule deyimindeki dosya yolu PHP modülünün bulunduğu yolu göstermelidir. make install komutu bunu sizin için yapmış bile olabilir. Emin olmak için kontrol edin.

    PHP 4 için: 
          LoadModule php4_module libexec/libphp4.so
    PHP 5 için: 
          LoadModule php5_module libexec/libphp5.so
  15. Ve httpd.conf dosyasının AddModule bölümünde ClearModuleList altında bir yere şu eklenir:

    PHP 4 için: 
          AddModule mod_php4.c
    PHP 5 için: 
          AddModule mod_php5.c
  16. Apache sunucuya hangi uzantıların PHP olduğu bildirilir. Örneğin: 
          AddType application/x-httpd-php .php .phtml
    PHP kaynak kodları için .phps kullanılabilir: 
          AddType application/x-httpd-php-source .phps
  17. Kurulum tamamlanınca Apache sunucuyu durdurup yeniden başlatmak gerekir.

Apache için statik modül kurulumu

  1. gunzip -c apache_1.3.x.tar.gz | tar xf -
  2. cd apache_1.3.x
  3. ./configure
  4. cd ..
  5. gunzip -c php-5.x.y.tar.gz | tar xf -
  6. cd php-5.x.y
  7. ./configure --with-mysql --with-apache=../apache_1.3.x
  8. make
  9. make install
  10. cd ../apache_1.3.x
  11. ./configure --prefix=/www --activate-module=src/modules/php5/libphp5.a (Yukarıdaki satır doğrudur! Evet, şu anda libphp5.a yok ama yaratılacak)
  12. make (httpd ikiliniz var Apache bin yoluna taşıyabilirsiniz. Eğer ilk kurulumunuzsa "make install" da kullanmalısınız)
  13. cd ../php-5.x.y
  14. cp php.ini-dist /usr/local/lib/php.ini
  15. /usr/local/lib/php.ini dosyasını PHP işlemleri için değiştirebilirsiniz. httpd.conf ya da srm.conf dosyalarını değiştirin ve aşağıdakini akleyin:
    AddType application/x-httpd-php .php

Yeniden Apache başlatmak için komutlar

  1. Birçok Linux ve SysV türlerinde:
    /etc/rc.d/init.d/httpd restart
  2. apachectl kodlamasını kullanarak:
    /path/to/apachectl stop
    /path/to/apachectl start
  3. httpdctl ve httpsdctl (OpenSSLkullanarak), apachectl benzeridir:
    /path/to/httpsdctl stop
    /path/to/httpsdctl start
  4. mod_ssl, ya da başka SSL sunucu kullanarak, elle durdurup başlatmak isteyebilirsiniz:
    /path/to/apachectl stop
    /path/to/apachectl startssl

Apache için değişik biçimlerde PHP derleme

   ./configure --with-apxs --with-pgsql
libphp5.so (PHP4 için libphp4.so) paylaşımlı kitaplık yaratır. Apache httpd.conf dosyasının LoadModule satırından yüklenir. Kitaplığa PostgreSQL desteği eklenmiştir. 
   ./configure --with-apxs --with-pgsql=shared
Apache için yaratılan 'libphp4.so' paylaşımlı kitaplıktır. Aynı zamanda 'pgsql.so' paylaşımlı kitaplığını da yaratılır. PHP yüklemesine uzatma olarak php.ini dosyasından yüklenir ya da kodlama içinden dl() işleviyle yüklenir. 
   ./configure --with-apache=/path/to/apache_source --with-pgsql
libmodphp5.a kitaplığını yaratır. mod_php5.c ve ek dosyaları bunu Apache kaynak ağacında src/modules/php5 yoluna kopyalar. Daha sonra Apache --activate-module=src/modules/php5/libphp5.a ile derlenir. Apache sistemi libphp5.a yaratır ve statik olarak httpd ikilisine eklenir (PHP4 için php5 yerine php4 yazılır). PostgreSQL desteği httpd ikilisinde doğrudan vardır. Sonuçta tek httpd ikilisi Apache ve tüm PHP kitaplıklarıyla birlikte yaratılır. 
   ./configure --with-apache=/path/to/apache_source --with-pgsql=shared
Aynı önceki gibi, PostgreSQL doğrudan desteği sonuç httpd ikilisine pgsql.so paylaşımlı kitaplığı yükleme olanağını getirir. Yükleme ya php.ini dosyasıyla ya da doğrudan dl() kullanılarak yapılır.

Değişik yollarda PHP kurmak istediğinizde, her yöntemin yararlarını ve zorluklarını düşünmelisiniz. Paylaşımlı nesne olarak yaratmanın anlamı Apache ayrıca derlenebilir demektir. Bu durumda değişikliklerde herşey yeniden derlenmez. Apache içine PHP gömülünce (statik yöntem) PHP daha hızlı yüklenir ve hızlı çalışır.

Not: Apache varsayılan httpd.conf parametre dosyası içindeki bir bölümünün aşağıda gösterildiği gibi olan satırları vardır: 
User nobody
Group "#-1"
Buradaki satır "Group nogroup" olarak değişmeden (ya da Group daemon) PHP dosyaları açamaz.

Not: --with-apxs=/apxs_icin/yol/apxs kullanımında apxs sürümünü doğru belirttiğinizde emin olun.

Apache 2 Paylaşımlı Modül Sürümü

  1. gzip -d httpd-2_0_NN.tar.gz
  2. tar xvf httpd-2_0_NN.tar
  3. gunzip php-NN.tar.gz
  4. tar -xvf php-NN.tar
  5. cd httpd-2_0_NN
  6. ./configure --enable-so
  7. make
  8. make install

    Şimdi /usr/local/apache2 altında Apache 2.0.NN var. MPM önceden çatallama ve yüklenebilir modüllere donatılmıştır. Apache denemek için:
    /usr/local/apache2/bin/apachectl start
    PHP ayarlarını tamamlamak için sunucu durdurulur:
    /usr/local/apache2/bin/apachectl stop.
  9. cd ../php-NN
  10. Şimdi PHP ayarlarını yapın. Burada PHP çeşitli seçeneklerle ayarlanır ./configure --help var olan seçenekleri belirtir. Örnek aşağıda verilmiştir: 
          ./configure --with-apxs2=/usr/local/apache2/bin/apxs --with-mysql
  11. make
  12. make install
    Eğer ayarları değiştirmek gerekirse yalnız son üç adım tekrarlanır.
  13. php.ini ayarlarını yapmak 
        cp php.ini-dist /usr/local/lib/php.ini
    PHP seçenekleri için .ini dosyası değiştirilir. php.ini dosyasının yerini değiştirmek için 10. adımda --with-config-file-path=/some/path kullanılır

    Eğer önerilen php.ini dosyasını kullanacaksanız nelerin değiştiğini okuyun.
  14. httpd.conf dosyasını php modülü için değiştirin. LoadModule komutunun sağ tarafındaki yol php modulünün yerini gösterir.

    PHP 4 için: 
          LoadModule php4_module modules/libphp4.so
    PHP 5 için: 
          LoadModule php5_module modules/libphp5.so
  15. Apache bazı uzantıları PHP olarak anlasın diye gereken değişiklikler yapılmalıdır. 
          AddType application/x-httpd-php .php .phtml
    PHP kaynaklarını göstermek için de .phps uzantısı kullanılır. 
          AddType application/x-httpd-php-source .phps
  16. Apache sunucuyu normal yollardan başlat 
          /usr/local/apache2/bin/apachectl start
    Apache SAPI modülü biçimindeki PHP desteğiyle çalıştırılır. Her ikisi için de bir çok ayar seçeneği vardır. Kaynak kod ağacı altında ./configure --help yazarak seçenekler gösterilir. "Multithread" sürümü yapmak için --with-mpm=worker ya da --with-mpm=perchild seçenekleriyle ayar yapılır.

    Not: "multithreaded" Apache sürümü için sisteminizin "thread" özelliğini desteklemesi gerekir. Ayrıca PHP "Zend Thread Safety" (ZTS) seçeneğiyle kurulmalıdır.

Caudium

PHP 4 Caudium web sunucusu için Pike modülü olarak kurulur.

Caudium Kurulum komutları şöyledir:
  1. PHP 4 kurulumuna başlamadan önce Caudium kurulu olmalıdır. PHP 4 düzgün çalışması için Pike 7.0.268 ya da daha yenisi olmalıdır. Kolaylık olsun diye Caudium /opt/caudium/server/ altına kurulmuş olsun.
  2. dosya yolunu php-x.y.z olarak değiştirin (burada x.y.z sürüm numaralarıdır).
  3. ./configure --with-caudium=/opt/caudium/server
  4. make
  5. make install
  6. Hala çalışıyorsa Caudium yeniden başlatılsın.
  7. Grafiksel ayar arayüzüne giriş yapın ve PHP 4 desteğinin ekleneceği sanal sunucuya gidin.
  8. Modül eklemeyi tıklayın ve PHP 4 kodlama desteği modülünü ekleyin.
  9. Eğer belge "PHP 4 yorumlayıcı yoktur" derse sunucuyu yeniden başlatmış olduğunuzu kontrol edin. /opt/caudium/logs/debug/default.1 PHP4.so hataları için denetlendiğinde caudium/server/lib/[pike-version]/PHP4.so varlığı incelenmelidir.
  10. Eğer gerek varsa PHP 4 kodlama desteği modülünü ayarlayın

    Caudium modülünü PHP 4 için var olan uzantılarla derleyebilirsiniz.

    Not: Eğer PHP 4 MySQL desteğiyle derlenirse normal MySQL istemci kodu kullanıldığına dikkat edin. Aksi halde Pike, MySQL desteği olduğundan hata verir. --with-mysql seçeneğiyle elde edersiniz.

fhttpd related notes

PHP yaratırken "fhttpd modülü olarak yaratılacak mı?" sorusuna "yes" denirse (ayarlamada --with-fhttpd=DIR seçeneği) ve fhttpd kaynak yolunu belirtilirse PHP fhttpd modülü olur. Varsayılan veri yolu /usr/local/src/fhttpd. Not: fhttpd için PHP 4.3.0 için destek vermez.

Sun Solaris üzerinde Sun, iPlanet ve Netscape sunucular

Buradakiler Sun Java System Web Sunucusuna ait Sun ONE Web Server, iPlanet ve Netscape sunucuya kurulacak PHP içindir.

PHP 4.3.3 ile PHP kodlama kullanarak NSAPI modülü yaratılır.

http://benoit.noss.free.fr/php/install-php4.html sayfasına bakılabilir.

Sun JSWS/Sun ONE WS/iPlanet/Netscape web sunucularıyla PHP kurmak için, --with-nsapi=[DIR] seçeneği ile kurulum yolu girilir. /opt/netscape/suitespot/ varsayılan yoldur. Lütfen /php-xxx-version/sapi/nsapi/nsapi-readme.txt dosyasını okuyun.
  1. Aşağıdaki paketleri yükleyin siteden http://www.sunfreeware.com/ yükleyin: 
           autoconf-2.13
       automake-1.4
       bison-1_25-sol26-sparc-local
       flex-2_5_4a-sol26-sparc-local
       gcc-2_95_2-sol26-sparc-local
       gzip-1.2.4-sol26-sparc-local
       m4-1_4-sol26-sparc-local
       make-3_76_1-sol26-sparc-local
       mysql-3.23.24-beta (if you want mysql support)
       perl-5_005_03-sol26-sparc-local
       tar-1.13 (GNU tar)
  2. PATH=.:/usr/local/bin:/usr/sbin:/usr/bin:/usr/ccs/bin olmalı ve sistemde PATH bu bilgileri içermelidir.
  3. gunzip php-x.x.x.tar.gz (.gz dağılımınız varsa yoksa 4. adıma gidin).
  4. tar xvf php-x.x.x.tar
  5. açılan PHP yoluna gidin: cd ../php-x.x.x
  6. Sonraki adımlar için Netscape /opt/netscape/suitespot/ iyolunda kurulmuş olsun. Yoksa yolu değiştirin. 
    ./configure --with-mysql=/usr/local/mysql \
--with-nsapi=/opt/netscape/suitespot/ \
--enable-libgcc
  7. make ve make install.

    İlk kurulumdan sonra ayarları yapabilirsiniz.

    Sun/iPlanet/Netscape için ayar komutları şöyledir. Önce LD_LIBRARY_PATH çevre değişkenine bazı yollar eklenir. Başlama kodlaması çoğu kez /path/to/server/https-servername/start yolunda olur. Ayarlama dosyalarını değiştirebilirsiniz /path/to/server/https-servername/config/.
  1. Aşağıdaki satırı mime.types dosyasına eklenmelidir. 
    type=magnus-internal/x-httpd-php exts=php
  2. magnus.conf dosyasını (sunucular <= 6 için) ya da obj.conf (sunucular < 6) dosaysını değiştirin. sisteminize göre shlib /opt/netscape/suitespot/bin/libphp4.so içerecek biçimde olur. Aşağıdaki mime tipleri ilklemesinden sonra şunlar eklenir: 
    Init fn="load-modules" funcs="php4_init,php4_execute,php4_auth_trans" shlib="/op
t/netscape/suitespot/bin/libphp4.so"
Init fn="php4_init" LateInit="yes" errorString="Failed to initialize PHP!" [php_
ini="/path/to/php.ini"]
    (PHP <= 4.3.3) php_ini parameterleri zorunlu değildir ama onunla web sunucu ayar yolundaki php.ini dosyasını değiştirirsiniz.
  3. obj.conf dosyasındaki varsayılan nesneleri değiştirilebilir. (sanal sunucularda dosya vserver.obj.conf olur): 
    <Object name="default">
.
.
.
.#NOT sonraki satır 'ObjectType' sonrasında ve 'AddLog' öncesinde olmalıdır.
Service fn="php4_execute"
        type="magnus-internal/x-httpd-php"
        [inikey=value inikey=value ...]
.
.
</Object>
    (PHP <= 4.3.3) ek parametreler olarak bazı özel php.ini-values ekleyebilirsiniz, Örneğin docroot="/path/to/docroot" php4_execute çalıştırabilirsiniz. Mantıksa ini-keys (ilkleme anahtarları) için 0/1 kullanın. "On","Off",... (BU doğru çalışmaya bilir), örneğin zlib.output_compression=1 yazılmalı
  4. Bir yolu ayarlamak isterseniz yalnız buna gerek vardır. 
    <Object name="x-httpd-php"<
ObjectType fn="force-type" type="magnus-internal/x-httpd-php"
Service fn=php4_execute [inikey=value inikey=value ...]
</Object<
    Daha sonra yönetim sunucusunda bir yolu ayarlayarak ve bunu x-httpd-php stiline eşleyerek tüm dosyaların PHP ile çalışması sağlanmış olur. PHP kullanımını gizleyerek .html biçiminde dosyaları belirtmek güzel görülebilir.
  5. Güvenlik ayarları: PHP güvenliği diğer güvenliklerle birlikte kullanılabilir. Tüm güvenlik PHP kodlamasına geçer. Varsayılan nesnenize aşağıdakileri eklemeniz gerekir: 
    <Object name="default"<
AuthTrans fn=php4_auth_trans
.
.
.
</Object<
  6. PHP güvenliğinin bir yolda olması için aşağıdaki eklenir: 
    <Object ppath="d:\path\to\authenticated\dir\*">
AuthTrans fn=php4_auth_trans
</Object>
    Not: PHP için kullanılan tampon boyutu (stacksize) web sunucu ayarlarına bağlıdır. Büyük PHP kdlamalarında kırılım olursa bu değerin değişmesi gerekebilir.

CGI çevresi ve php.ini içinde yapılması gereken değişiklikler

PHP kodlaması yaparken dikkat edilmesi gereken Sun JSWS/Sun ONE WS/iPlanet/Netscape "multithread" web sunucudur Bu nedenle tüm istekler aynı işlem uzayında (web sunucunun kendi uzayı) çalışır. Eğer PATH_INFO, HTTP_HOST gibi CGI değişkenlere erişmek isterseniz bu uzay tek çevre ortamındadır ve eskiden olduğu gibi getenv() kullanmak gerekebilir. Ya da $_ENV evrensel değişken kullanılır.

Not: Çevrede neden geçersiz CGI değişkenleri vardır?

Yanıt: Çünkü web sunucu işlemi admin sunucusu olarak başlar (bir CGI kodlama - admin sunucusundaki CGI kodlamasıdır - çevre değişkenlerini içerebilir. Admin sunucusundan başlatmazsanız bu çevre değişkenlerine erişemezsiniz.

Kodlamasınız CGI değişkenlerine erişmesi için $_SERVER super evrensel değişken kullanılır. 
variables_order = "GPCS"
register_globals = On

Hatalı sayfalar için özel kullanım ya da
kendi kendine yapılan klasör listesi (PHP >= 4.3.3)

PHP "404 Not Found" (404 Bulunamadı) türü hata sayfaları yapmak için de kullanılır. Değiştirmek istediğiniz her sayfa için aşağıdaki kodlamayı obj.conf dosyasına ekleyin. 
Error fn="php4_execute"
         code=XXX script="/path/to/script.php"
         [inikey=value inik ey=value...]
Burada XXX HTTP hata kodudur. Lütfen başka hata kodu yönlendiricisi varsa sorun olmaması için silin. Sizin kodlamanız $_SERVER['ERROR_TYPE'] ile HTTP durum koduna erişebilir.

Bir diğer özellik ise kendi kendine üretilen klasör listesidir. Klasör listesi üreten PHP kodlaması yaratılır ve varsayılan servis satırını type="magnus-internal/directory" iken aşağıdakiyle değiştirilir (obj.conf):

Service fn="php4_execute" type="magnus-internal/directory" script="/path/to/script.php" [inikey=value inikey=value...]

Hata ve klasör listesi sayfaları için URL adresleri $_SERVER['PATH_INFO'] ve $_SERVER['PATH_TRANSLATED'] değişkenlerindedir.

nsapi_virtual() ve alt istekler için notlar (PHP >= 4.3.3)

NSAPI modülü şimdi nsapi_virtual() işlevini destekler (eşdeğeri: virtual()). Web sunucu için alt istekler yapar ve sonuçları web sayfasına ekler. Bu özellik belgelenmemiş NSAPI kitaplığı yeteneklerini içerir. Unix ortamında modül otomatik olarak istenen işlevlere bakar ve varsa onları kullanır. Yoksa nsapi_virtual() kapatılır.

Not: nsapi_virtual() desteği hala deneyseldir.

CGI ve komut satırı ayarları

Varsayılanlarla PHP, CGI programı olarak kurulur. Böylece PHP CGI işlemi olarak kullanılabilir ya da web-ilişkili olmayan PHP kodlaması olarak kullanılabilir. Eğer Web sunucu çalışıyorsa ve PHP modül desteği veriyorsa, siz daha çok bu çözümü kullanırsınız. Ama CGI sürümü kullanıcıların açılmış değişik PHP sayfaları değişik kullanıcı kimlikleriyle kullanmalarını sağlar.

Uyarı

CGI ayarı kullanarak, sunucunuz çeşitli saldırılara açık olur.

PHP 4.3.0 sürümünden bu yana PHP koşullarında önemli değişiklikler oldu. CLI adlı yeni SAPI vardır ve aynı CGI ikilileri gibidir. {PREFIX}/bin/php yoluna neyin kurulduğu ayar satırınıza bağlıdır.

OpenBSD kurulum notları

OpenBSD 3.6 üzerine kurulacak PHP ile ilgili notları içerir.

İkili pakaetleri kullanmak

İkili paket kullanarak OpenBSD hem önerilendir hem de en kolay kurulumdur. Çekirdek paket birçok modüle bölünmüştür ve her biri bağımsız olarak kurulabilir ya da kaldırılabilir. Gereken dosyalar ya OpenBSD CD dağılımında ya da Ftp sitelerinde vardır.

Kurulmak istenecek ana paket php4-core-4.3.8.tgz tar topudur ve temel makineyi içerir (artı gettxt ve iconv). Daha sonra modül paketlerine geçilir. Örneğin: php4-mysql-4.3.8.tgz ya da php4-imap-4.3.8.tgz. php.ini dosyasından bu modüller phpxs komutuyla etkinleştirilir ya da kaldırılır.

OpenBSD paket kurulumu: 
# pkg_add php4-core-4.3.8.tgz
# /usr/local/sbin/phpxs -s
# cp /usr/local/share/doc/php4/php.ini-recommended /var/www/conf/php.ini
  (mysql ekleme)
# pkg_add php4-mysql-4.3.8.tgz
# /usr/local/sbin/phpxs -a mysql
  (imap ekleme)
# pkg_add php4-imap-4.3.8.tgz
# /usr/local/sbin/phpxs -a imap
  (test için mysql kaldır)
# pkg_delete php4-mysql-4.3.8
# /usr/local/sbin/phpxs -r mysql
  (PEAR kitaplıklarını kur)
# pkg_add php4-pear-4.3.8.tgz
OpenBSD ikili paketleri konusunda packages(7) kitapcığına bakınız.

Portların kullanımı

OpenBSD kullanmayı iyi bilen kişilere önerilecek bir başka yöntem port ağacını kullanarak PHP kaynak kodlarından derleme yapılmasıdır. PHP 4 port, iki alt klasöre bölünür: çekirdek (core) ve uzantılar (extensions). Uzantılan klasörü, desteklenen tüm PHP modülleri için alt paketler üretir. Eğer bu modüllerin bazılarını kullanmak istemezseniz no_* FLAVOR ile belirtirsiniz. Örneğin imap modülünü atlamak için FLAVOR değeri no_imap yapılır.

Solaris için özel kurulum ipuçları

Bu bölüm Solaris Sistemlerine kurulacak PHP için özel notları içerir.

Gerekli yazılım

Solaris çoğunlukla "lack C" derleyicisi ve onun ilişkili araçlarını kullanır. Halbuki bu araçların bazıları için GNU sürümlerine gereksinim vardır. Gerekli yazılımlar şöyledir: 
     * gcc (recommended, other C compilers may work)
     * make
     * flex
     * bison
     * m4
     * autoconf
     * automake
     * perl
     * gzip
     * tar
     * GNU sed
Bunlara ek olarak ayarlarınıza uygun başka ek yazılımlar da kurmak (ve derlemek) istersiniz (Oracle ya da MySQL gibi).

Paketlerin kullanımı

Solaris kurulum işlemlerini pkgadd kullanarak kolaylaştırabilirsiniz. Bu araç gerekli parçaların da kurulumunu yapar.

Debian GNU/Linux kurulum notları

Bu bölüm Debian GNU/Linux üzerine PHP kurmaya özel ip uclarını ve notları içerir.

APT Kullanımı

Yalnız PHP kaynağını indirip derleyebilirsiniz. Debian paketleme sistemi en temiz PHP kurulum biçimidir. Eğer Linux üzerinde yazılım kurmayı bilmiyorsanız bu yolu deneyebilirsiniz.

İlk verilecek karar hangi Apache sürümünün kullanılacağıdır: Apache 1.3.x ya da Apache 2.x. Bunlarla ilintili PHP paketleri sırasıyla libapache-mod-php* ve libapache2-mod-php* olarak adlandırılmıştır. Debian üzerinde PHP 5 için resmi kurulum adımı yoktur. Bunlar hep PHP 4 içindir.

Debian için CGI ve CLI biçiminde PHP vardır. Sırasıyla php4-cgi ve php4-cli diye adlandırılır. Kurulacak başka bir özel paket, php4-pear olur. En küçük PEAR kurulumunu ve pear komut satırı hazır programarını içerir.

PHP için durağan ve daha yeni paketleri kurmak isterseniz http://www.apt-get.org/ URL adresine bakmalısınız. Bu resmi olmayan site, Debian paketlerinin bakımını yapar. Bunun için /etc/apt/sources.lists dosyasına aşağıdaki satırları ekleyin ve apt-get çalıştırarak güncellemelere erişin:

İki Dotdeb ilişkili satırlar 
deb http://packages.dotdeb.org stable all
deb-src http://packages.dotdeb.org stable all
En son düşünülecek şey, listelediğiniz paketler güncel mi? Son zamanlarda güncellemediyseniz apt-get çalıştırarak herşeyden önce güncelleme yapabilirsiniz. Bu biçimiyle en son sürüm Apache ve PHP kullanıyor olursunuz.

Herşey yerli yerinde olunca, aşağıdaki örneği Apache ve PHP kurulumu için kullanabilirsiniz.

Apache 1.3 ile Debian kurulum örneği 
   # apt-get install libapache-mod-php4
APT otomatik olarak Apache 1.3 için PHP 4 modülünü ve bağlı olduklarını kurar ve etkinleştirir. Eğer kurulumda Apache yeniden başlasın demediyseniz bunu elinizle yapmalısınız:

PHP kurulduktan sonra Apache durdurup başlatmak. 
# /etc/init.d/apache stop
# /etc/init.d/apache start

Ayarlarda daha iyi denetim

Son bölümde PHP yalnız çekirdek modüllerle kurulmuştur. Bu sizin istediğiniz olmayabilir ve MySQL, cURL, GD vb modülleri de istiyor olabilirsiniz.

Eğer kaynak kodlardan PHP derliyorsanız seçimlerinizi komut satırında "configure" komutuyla yaparsınız. APT ile yeni paketler eklenir 'php4-*' gibi (ya da 'php5-*' eğer PHP 5 kuruyorsanız).

PHP ek paketlerinin listesini almak 
   # dpkg -l 'php4-*'
Son çıktıdan da görebildiğiniz gibi, kurulabilecek çok fazla PHP modülü vardır. Aralarından seçim yaparak kurabilirsiniz. APT bağlantıları otomatik bulur ve gerekenleri kurar.

Eğer MySQL, cURL ve GD desteği eklemeyi seçerseniz, komut şöyle olur:

MySQL, cURL ve GDA ile PHP kurma 
   # apt-get install php4-mysql php4-curl php4-gd
APT otomatik olarak php.ini dosyanıza gereken satırları ekler (/etc/php4/apache/php.ini, /etc/php4/cgi/php.ini,vb). Bu satırlar iPHP içinde MySQL, cURL ve GD ietkinleştirir 
extension=mysql.so
extension=curl.so
extension=gd.so
Artık yapılacak şey modülleri etkinleştirmek için Apache durdurup başlatmaktır.
